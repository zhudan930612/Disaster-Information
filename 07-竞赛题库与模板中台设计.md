# 灾害信息员大赛平台 · 竞赛题库与模板中台设计

## 1. 文档定位

本文档用于明确大赛平台的题型定义、题目数据模型、题库分层结构、组题策略、题库与模板关系、版本管理和权限边界，作为练习赛模块、正式赛事模块、赛事管理后台和学员端设计的上游约束。

本文档重点解决以下问题：

- 平台支持哪些题型，每种题型的数据结构是什么
- 题库如何分层，平台共用题库和省级私有题库的边界在哪里
- 练习赛和正式比赛分别如何组题，谁可以配置组题规则
- 题库版本如何管理，历史比赛记录如何与题目变更解耦
- 不同角色对题库的访问权限边界在哪里

本文档不展开页面交互细节和接口字段清单，后续在模块功能设计和接口设计中展开。

---

## 2. 设计目标

- 支撑练习赛和正式比赛两类业务场景的题目供给
- 通过题库分层和装载绑定机制吸收省级差异，降低交付成本
- 通过组题策略配置而非手工指定，保证题目分配的公平性和可追溯性
- 通过版本锁定机制，确保历史比赛记录可复盘，不受题目后续变更影响
- 通过权限边界，防止省级获取超出授权的题库内容

---

## 3. 设计原则

- **题型固定**：当前仅支持判断、单选、多选三种题型，不建设自由题型扩展引擎
- **组题策略化**：具体题目由策略规则生成，不由省级逐题手工指定
- **题库分层**：平台共用题库和省级私有题库两层，互不自动共享
- **版本锁定**：比赛开始后锁定组题结果，题目内容变更不影响进行中和已结束的比赛
- **权限最小化**：省级管理员可配置题库使用范围，但不能查看不属于本省范围的题目内容

---

## 4. 题型定义

平台当前支持以下三种题型：

### 4.1 判断题

- 题干描述一个陈述，学员判断正误
- 选项固定为两个：正确 / 错误
- 有且仅有一个正确答案

### 4.2 单选题

- 题干提出一个问题
- 选项数量建议为 4 个（A / B / C / D），由内容方录入时确定
- 有且仅有一个正确答案

### 4.3 多选题

- 题干提出一个问题
- 选项数量建议为 4 至 5 个（A / B / C / D / E），由内容方录入时确定
- 正确答案数量为 2 个及以上
- 学员必须全部选对才算答对，不支持"选对几个算几分"的半分机制

---

## 5. 题目数据模型

每道题目至少包含以下字段：

| 字段 | 说明 |
|------|------|
| 题目 ID | 全局唯一标识 |
| 题型 | 判断 / 单选 / 多选 |
| 题干 | 题目描述正文，支持纯文本 |
| 选项列表 | 判断题固定 2 项；单选题 2-4 项；多选题 2-5 项 |
| 正确答案 | 判断题为单值；单选题为单值；多选题为多值集合 |
| 解析 | 答题结束后展示的解题说明，必填 |
| 难度等级 | 建议分三级：基础 / 中等 / 较难 |
| 知识点标签 | 支持多标签，用于组题策略筛选 |
| 题库归属 | 平台共用 / 省级私有（含所属省份标识） |
| 使用场景 | 练习赛 / 正式比赛 / 通用（可多选） |
| 状态 | 草稿 / 待审核 / 已启用 / 已停用 |
| 版本号 | 题目内容变更时递增 |
| 创建人 | 操作人标识 |
| 创建时间 | - |
| 最近修改时间 | - |

说明：

- 题干当前不支持图片、公式等富文本，如后续有需要，在功能设计阶段单独确认
- 选项内容当前为纯文本
- 解析字段为必填项，上线前由内容方完成补录

## 6. 题库分层结构

### 6.1 平台共用题库

- 由平台内容运营团队统一维护
- 供所有已装载的租户在规则允许范围内使用
- 适用于通用知识点、标准赛事场景
- 省级管理员不能修改、删除共用题库内的题目

### 6.2 省级私有题库

- 支持两种录入方式：由我方运营人员代录，或由省级管理员自助上传（省级自审）；归属于指定租户
- 仅服务于本省的练习赛和正式赛事配置
- 不自动向其他租户开放

### 6.3 题库装载与绑定

- 租户开通时由我方运营人员完成题库装载，绑定该租户可使用的题库范围
- 题库范围绑定后，省级管理员在配置赛事时只能在已装载的题库范围内选择
- 题库范围调整需由我方运营人员操作，省级管理员不能自行扩展
- 题库版本管理见第 8 节

## 7. 组题策略

### 7.1 策略原则

- 题目由策略规则自动生成，省级不能逐题手工指定
- 策略配置属于配置中台范畴，省级可在允许范围内调整配置参数
- 同一场次内所有参赛用户使用相同的题目集合（同卷机制），以保证公平性
- 同卷机制：在赛事开始前，由系统按策略生成当场次题目集合并锁定，所有用户答同一套题

### 7.2 练习赛组题策略

练习赛每次答题从练习题库中按以下参数生成题目：

| 配置项 | 说明 | 配置方 |
|--------|------|--------|
| 总题量 | 每次练习的题目总数 | 平台运营人员（模板级配置） |
| 各题型比例 | 判断 / 单选 / 多选的题数分配 | 平台运营人员（模板级配置） |
| 难度分布 | 各难度等级的题数分配 | 平台运营人员（模板级配置） |
| 知识点覆盖 | 是否要求覆盖指定知识点范围 | 平台运营人员（可选配置） |
| 题目去重规则 | 近期已答题目的回避策略 | 平台运营人员（模板级配置） |

说明：

- 练习赛不采用同卷机制，每次练习独立随机抽题
- 抽题结果由系统按策略生成，不向用户提前展示题目列表
- 去重机制已确认启用：近 N 天内用户已答过的题目不重复出现，N 值待确认
- 当题库可用题目数量不足以满足去重要求时，允许已答题目重复出现，系统不阻止组题

### 7.3 正式赛事组题策略

正式赛事每个阶段（初赛 / 复赛 / 决赛）从比赛题库中按以下参数生成题目：

| 配置项 | 说明 | 配置方 |
|--------|------|--------|
| 总题量 | 该阶段题目总数 | 省级管理员（在平台设定范围内配置） |
| 各题型比例 | 判断 / 单选 / 多选的题数分配 | 省级管理员（在平台设定范围内配置） |
| 难度分布 | 各难度等级的题数分配 | 省级管理员（在平台设定范围内配置） |
| 题库范围 | 共用题库 / 省内私有题库 / 混合 | 省级管理员（在已装载范围内选择） |
| 知识点范围 | 是否限定知识点标签范围 | 省级管理员（可选配置） |

说明：

- 正式赛事采用同卷机制：赛事发布后、开赛前，由系统按策略生成该阶段题目集合并锁定
- 组题结果锁定后，赛事进行中不允许修改题库范围和组题策略
- 每个阶段独立生成题目集合，不跨阶段复用

### 7.4 可配置范围边界

省级不能配置以下内容：

- 组题算法底层逻辑
- 同卷机制是否启用
- 跨租户题库的使用
- 超出装载范围的题库

## 8. 题库版本管理

### 8.1 版本管理原则

- 题目内容（题干、选项、答案、解析）发生变更时，版本号递增
- 历史比赛记录绑定该场次使用的题目版本快照，不受后续题目变更影响
- 比赛开始后，当场次题目集合以快照形式锁定，即使原始题目被修改或停用，该场次结果仍可复盘

### 8.2 快照时机

- 练习赛：每次答题开始时生成该次题目集合快照
- 正式赛事：赛事发布后、开赛前完成题目集合生成，并在比赛开始时锁定为快照

### 8.3 快照内容

每次比赛场次快照至少包含：

- 场次标识
- 题目 ID 列表
- 各题目的版本号
- 组题策略参数快照
- 快照生成时间

### 8.4 题目停用规则

- 已停用题目不参与新的组题
- 已停用题目仍可在历史比赛记录中查看
- 比赛进行中不允许停用已进入当场次快照的题目

---

## 9. 模板与题库关系

| 业务场景 | 使用题库 | 组题来源 |
|----------|----------|----------|
| 练习赛 | 练习题库 | 从使用场景标记为"练习赛"或"通用"的题目中抽取 |
| 正式赛事（初赛 / 复赛 / 决赛） | 比赛题库 | 从使用场景标记为"正式比赛"或"通用"的题目中抽取 |

说明：

- 同一道题目可同时标记为"练习赛"和"正式比赛"使用场景
- 练习赛题库与正式赛事题库通过使用场景标签区分，不是物理隔离的两个库
- 省级可在配置时选择是否允许练习赛题目与正式赛事题目交叉，但交叉使用需在策略中明确配置

## 10. 权限边界

| 角色 | 可操作范围 | 不可操作范围 |
|------|-----------|-------------|
| 平台内容运营 | 平台共用题库录入、编辑、审核、停用；题型配置；知识点标签管理 | 不直接参与省级私有题库审核与管理 |
| 我方运营人员 | 题库装载与绑定；省级私有题库协助录入；题库范围调整；版本管理操作 | 不替省级管理员承担赛事中的题库参数配置 |
| 省级主管管理员 | 在已装载范围内选择题库范围；配置各阶段组题参数 | 不能查看其他省私有题库内容；不能逐题手工指定；不能修改共用题库 |
| 地市县执行管理员 | 无题库配置权限，可查看已发布赛事的组题说明 | 不具备任何题库管理能力 |
| 国家级管理员 | 无题库配置或查看权限（仅查看统计结果） | 不具备任何题库访问能力 |
| 平台超级管理员 | 平台级题库策略配置；跨租户题库治理；内容安全审计 | 不直接替省级操作赛事配置 |

## 11. 题库质量与内容管理

### 11.1 上线前基本要求

题目上线前需满足：

- 题干表述完整，无歧义
- 所有选项填写完整
- 正确答案已标注
- 解析已填写
- 难度等级已标注
- 知识点标签已关联至少一个

### 11.2 停用条件

以下情况题目应被停用，不参与新的组题：

- 题干或答案存在明显错误
- 题目已过期（政策变化、内容失效）
- 内容重复度过高

### 11.3 内容运营要求

- 平台应保持各题型、各难度等级的题目数量充足，避免组题时题量不足
- 建议共用题库在各主要知识点分类下保持足够题目储备
- 题库充足性由内容运营团队定期评估，评估结果可通过管理后台数据呈现

## 12. 对其他模块的约束

### 12.1 对练习赛模块

- 练习赛每次开始时，必须从练习题库中按组题策略生成题目集合
- 用户不能选择具体题目，只能触发组题
- 答题结束后可展示本次题目及解析，但不保留题目列表供下次提前查看

### 12.2 对正式赛事模块

- 正式赛事的每个阶段在发布后、开赛前必须完成题目集合生成与锁定
- 赛事进行中不允许修改题库范围和组题策略
- 统计模块中的成绩结果必须与该场次快照绑定，保证可复盘

### 12.3 对赛事管理后台

- 后台题库范围选择组件必须只展示已装载至该租户的题库
- 组题参数配置界面必须在平台设定的配置边界内展示可调范围，不可超出

### 12.4 对统计模块

- 统计模块应能关联题目快照，支持查看该场次的组题策略和题目组成（管理员视角）
- 不向学员展示其他用户的答题情况或题目命中情况

### 12.5 对数据配置中台

- 题库装载与范围绑定由配置中台统一管理，不由赛事管理后台独立配置
- 题库版本管理结果应在配置中台可查

## 13. 当前待确认事项

- 练习赛去重窗口期：已确认启用去重机制，具体近多少天内的已答题目不重复出现，N 值待确认

## 14. 结论

竞赛题库与模板中台是练习赛和正式赛事两类核心业务的供题底座。只有把题型定义、数据模型、题库分层、组题策略、版本管理和权限边界定清，后续练习赛模块、正式赛事模块、赛事管理后台和统计模块才能在一套一致的题目供给逻辑上稳定展开。
