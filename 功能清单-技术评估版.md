# 灾害信息员竞赛 SaaS 平台 — 功能清单（技术评估版）

> 生成日期：2026-04-27
> 用途：供技术团队评估各功能开发工时
> 依据：产品上下文、产品规划、各模块功能设计文档

---

## 说明

### 复杂度定义

| 复杂度 | 判定标准 | 预估参考（人天） |
|--------|----------|------------------|
| 高 | 涉及跨系统对接、复杂业务规则、需要新建核心表结构或算法 | 8-15 |
| 中 | 在现有能力上做适配改造、新增配置页或列表页、较复杂交互 | 4-8 |
| 低 | 简单配置项、文案调整、页面微调、通用列表/表单 | 1-3 |

### 建设策略定义

| 策略 | 说明 |
|------|------|
| 复用 | 国家平台已有成熟实现，本项目直接复用/移植，改动量小 |
| 适配 | 国家平台有参考能力，但需按省级 SaaS 口径做规则调整、字段改造、权限适配 |
| 新增 | 国家平台无现成能力，需全新设计开发 |

## 技术评估建议

1. **优先评估"新增"类功能**：这些是国家平台无直接参考的能力，研发不确定性最高，建议技术团队优先进行方案设计和工时评估。

2. **跨系统对接是最大风险点**：
   - 培训平台 ↔ 竞赛平台：用户同步、学习行为同步、练习次数发放（B-205、B-206、B-511）
   - 需尽早确认接口协议、同步频率、幂等策略、异常补偿机制

3. **省份隔离是架构核心**：
   - 数据层面：所有业务表需考虑省份归属字段和查询索引
   - 接口层面：所有查询接口需按角色注入省份过滤条件
   - 建议技术团队先设计省份隔离的通用方案，再落实到各模块

4. **复用能力的真实工作量**：
   - 虽然标注为"复用"，但国家平台代码不一定能直接移植，需评估代码抽取、环境适配、依赖解耦的工作量
   - 建议技术团队对"复用"类功能逐个确认：是直接复用代码 / 参考重写 / 还是只需配置调整

5. **前台页面现状**：
   - 练习、竞赛、我的页面结构已完成，但功能层面仍需重新开发并接通数据
   - 建议确认前端框架是否与国家平台一致，以判断组件复用程度
