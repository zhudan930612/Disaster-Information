# 灾害信息员大赛平台 · 学员竞赛小程序页面级功能设计

## 1. 文档定位

本文档承接《08-产品功能清单》，对学员竞赛小程序内每个页面的功能区块、核心交互逻辑、关键字段与状态、边界条件进行展开，作为原型设计和开发排期的直接依据。

本文档不输出视觉稿和原型图，后续由设计师基于本文档出原型。

**约束文档参考：**

- 04-国家级平台协同与数据同步设计（登录、联动、次数）
- 05-赛制模板与配置边界设计（赛事规则、阶段、门槛）
- 06-多租户与数据配置中台设计（权限、租户隔离）
- 07-竞赛题库与模板中台设计（题型、组题规则、去重）

## 2. 整体导航结构

小程序采用底部 Tab 导航，共四个一级入口：

| Tab | 名称 | 核心职责 |
|-----|------|----------|
| Tab 1 | 首页 | 练习入口、赛事概览、次数状态、联动引导 |
| Tab 2 | 练习 | 发起练习赛、查看练习历史和错题 |
| Tab 3 | 赛事 | 查看和参与正式赛事 |
| Tab 4 | 我的 | 个人信息、次数明细、参赛历史 |

## 3. 登录与身份

### 3.1 进入流程

**触发场景：** 用户直接打开小程序，使用国家级平台账号和密码登录。

**流程：**

1. 打开小程序，展示登录页
2. 用户输入国家级平台账号和密码
3. 校验通过 → 换取本平台登录态
4. 首次登录 → 自动建立身份映射 → 进入首页
5. 非首次且会话有效期内再次打开 → 直接进入首页（跳过登录页）

**字段：**

- 国家级平台账号
- 密码
- 本平台 Session Token（换取后本地缓存）
- 身份映射状态（已映射 / 首次映射中 / 映射失败）

**边界条件：**

| 情况 | 处理方式 |
|------|----------|
| 账号或密码错误 | 展示"账号或密码错误，请重试" |
| 身份映射失败 | 展示"账号同步中，请稍后再试"，记录异常日志，不拦截用户查看已有数据 |
| 国家级平台暂时不可用 | 若本地已有有效 Session，允许进入查看历史数据；联动相关功能降级提示 |

## 4. 首页

### 4.1 页面定位

用户进入小程序后看到的第一个页面，承担快速触达练习赛、了解当前赛事状态、感知次数余量、获取学习联动引导的职责。

### 4.2 功能区块

**区块一：练习赛快捷入口**

- 展示剩余答题次数（基础次数 + 奖励次数合计）
- 「开始练习」按钮
- 次数充足时：点击直接进入练习答题页
- 次数为 0 时：按钮状态变为「去完成学习任务」，点击跳转学习联动引导页

**区块二：正式赛事状态卡片**

- 展示本省当前开通的正式赛事（最多展示当前进行中或最近一场）
- 赛事名称、当前阶段、阶段状态（报名中 / 进行中 / 已结束）、截止时间
- 点击进入赛事详情页
- 无正式赛事时：该区块不展示或展示占位文案"暂无进行中的赛事"

**区块三：学习联动任务引导**

- 展示当前可通过完成学习任务获取次数的提示
- 展示可获得的次数奖励说明
- 点击跳转国家级平台（小程序间跳转或 H5 跳转）
- 次数充足时该引导仍可见但非强调状态；次数为 0 时突出展示

**区块四：赛事公告**

- 展示管理员发布的最新通知（最多 1-2 条）
- 超出时提供「查看更多」入口

### 4.3 数据来源

| 数据 | 来源 |
|------|------|
| 剩余次数 | 本平台用户权益账户 |
| 正式赛事信息 | 当前租户已发布赛事 |
| 学习任务信息 | 联动规则配置 |
| 公告 | 管理后台发布内容 |

## 5. 练习模块

### 5.1 练习模块首页

**页面定位：** 练习 Tab 的默认落地页，提供发起练习赛、查看历史记录和错题的入口。

**功能区块：**

- 次数余量展示（基础次数 / 奖励次数分开展示）
- 「开始练习」按钮（次数不足时变为引导态）
- 练习历史入口（展示最近几次记录摘要）
- 错题记录入口（展示错题总数）
- 练习排行榜入口

**边界条件：**

| 情况 | 处理方式 |
|------|----------|
| 次数为 0 | 「开始练习」变为「次数不足，去获取」，点击进入学习联动引导页 |
| 无练习历史 | 练习历史区块展示空状态文案 |

---

### 5.2 学习联动引导页

**页面定位：** 用户次数不足时的专项引导页，引导用户前往国家级平台完成学习任务以获取额外次数。

**功能区块：**

- 当前次数余量展示（基础 + 奖励）
- 说明文案：完成指定学习任务可获得额外练习次数
- 当前可完成的学习任务列表（来自联动规则配置）
  - 任务名称
  - 可获得次数
  - 是否已完成
- 「前往国家级平台」按钮（跳转至国家级平台）
- 返回练习的入口

**返回时的状态刷新：**

- 用户从国家级平台返回后，自动触发次数账户刷新
- 若次数已到账：展示"恭喜，已获得 N 次练习机会"提示，可直接开始练习
- 若次数未到账：展示"学习记录同步中，次数稍后到账"提示

---

### 5.3 练习答题页

**页面定位：** 用户发起一次练习赛后的答题交互页面。

**进入条件：** 剩余次数 > 0，扣减 1 次后进入，系统按组题策略生成本次题目集合。

**功能区块：**

**顶部信息栏：**
- 当前题号 / 总题数（如 3/20）
- 已用时长计时器（正计时，持续显示）

**题目区域：**
- 题干文本
- 题型标识（判断 / 单选 / 多选）
- 选项列表
  - 判断题：「正确」「错误」两个选项
  - 单选题：A / B / C / D 选项，单选
  - 多选题：A / B / C / D / E 选项，多选（可选 2 个及以上）

**操作区域：**
- 「下一题」按钮（选择答案后可点击）
- 最后一题变为「提交」按钮
- 不支持返回修改已作答的题目

**核心交互逻辑：**

1. 用户选择选项后，「下一题」按钮激活
2. 多选题需至少选择 2 个选项后才激活按钮
3. 点击「下一题」，题目切换，不展示当题是否答对
4. 点击「提交」，进入结果页

**边界条件：**

| 情况 | 处理方式 |
|------|----------|
| 用户中途退出 | 弹出确认框："退出后本次练习记录将不保存，确认退出？"；次数已扣减不退回 |
| 网络异常 | 提示"网络不稳定，请检查网络后重试"；题目内容已加载则允许继续作答 |

---

### 5.4 练习结果页

**页面定位：** 本次练习赛结束后的成绩和解析展示页。

**功能区块：**

**成绩摘要区：**
- 本次得分（如 85 分）
- 答对题数 / 总题数（如 17/20）
- 正确率（如 85%）
- 本次用时

**逐题解析区：**
- 按题目顺序展示每道题
- 展示：题干、选项、用户作答（标记对错）、正确答案、解析文本
- 答对的题目和答错的题目用颜色区分

**操作区：**
- 「再练一次」按钮（若次数 > 0 则直接发起；若次数为 0 则引导去获取）
- 「返回首页」按钮

**数据写入：**
- 本次练习记录写入历史
- 答错题目写入错题记录
- 练习成绩计入排行快照

---

### 5.5 练习历史页

**页面定位：** 展示用户历史练习记录。

**列表项字段：**
- 练习日期和时间
- 本次得分
- 正确率
- 用时

**支持操作：**
- 点击单条记录，展示该次的逐题解析详情（复用练习结果页解析区的样式）

**边界条件：**
- 无历史记录时展示空状态提示

---

### 5.6 错题记录页

**页面定位：** 汇总用户所有练习赛中答错的题目。

**列表展示：**
- 题干摘要（截断显示）
- 题型标识
- 最近一次答错时间

**点击展开：**
- 完整题干
- 选项
- 用户最近一次作答
- 正确答案
- 解析

**边界条件：**
- 无错题时展示空状态提示

---

### 5.7 练习排行榜页

**页面定位：** 展示本省练习赛的成绩排行情况。

**排行维度：**
- 按练习总得分排行（历史累计）

**列表字段：**
- 排名
- 姓名（脱敏展示中间部分）
- 所属单位
- 总得分 / 平均正确率

**我的位置：**
- 列表顶部或底部固定展示本人当前排名位置

---

## 6. 赛事模块

### 6.1 赛事列表页

**页面定位：** 展示用户可参与或正在参与的正式赛事。

**列表项字段：**
- 赛事名称
- 赛事类型（个人赛 / 团体赛）
- 当前活跃阶段（初赛 / 复赛 / 决赛）
- 阶段状态（报名中 / 进行中 / 已结束 / 未开始）
- 阶段截止时间
- 我的参与状态（未参与 / 已参与 / 已晋级 / 未晋级）

**边界条件：**

| 情况 | 处理方式 |
|------|----------|
| 无正式赛事 | 展示空状态："本省暂无正式赛事" |
| 本省未开通正式赛事 | 同上 |
| 赛事全部已结束 | 展示历史赛事，标注"已结束" |

---

### 6.2 赛事详情页

**页面定位：** 展示一场赛事的完整信息，以及用户进入答题的入口。

**功能区块：**

**赛事基本信息：**
- 赛事名称
- 赛事类型（个人赛 / 团体赛）
- 主办单位（省级名称）
- 赛事公告说明

**阶段信息列表（初赛 / 复赛 / 决赛）：**
- 各阶段时间（开始 - 结束）
- 题量
- 答题时长
- 晋级比例或晋级人数
- 阶段状态

**参赛门槛说明：**
- 当前阶段的参赛资格要求（文字说明）

**我的参与状态：**
- 当前阶段：是否已参与、成绩、排名、晋级状态
- 历史阶段：结果摘要

**团体赛补充信息（仅团体赛展示）：**
- 我所属团队名称
- 团队当前排名

**操作按钮：**
- 当前阶段进行中且未参与 → 「进入答题」
- 当前阶段进行中且已参与 → 「查看成绩」
- 当前阶段未开始 → 「未开始，敬请期待」（不可点击）
- 当前阶段已结束 → 「查看结果」

---

### 6.3 参赛资格校验（前置拦截）

**触发时机：** 用户点击「进入答题」后，系统在进入答题页前完成校验。

**校验项（按省级配置启用）：**

| 校验项 | 不通过时提示文案示例 |
|--------|---------------------|
| 是否为本省用户 | "本赛事仅限本省灾害信息员参与" |
| 是否满足在岗状态 | "您当前的在岗状态不满足参赛要求" |
| 是否完成指定练习次数 | "参赛需完成至少 N 次练习，当前已完成 M 次" |
| 是否在指定组织范围内 | "您当前不在本赛事开放的参赛范围内" |

**校验通过：** 直接进入赛事答题页。

**校验不通过：** 展示拦截弹窗，说明具体原因，提供「我知道了」关闭按钮。

---

### 6.4 赛事答题页

**页面定位：** 正式赛事的答题交互页面，规则与练习赛答题页类似，但有以下差异。

**与练习答题页的差异：**

| 项目 | 练习赛 | 正式赛事 |
|------|--------|----------|
| 计时方式 | 正计时（已用时长） | 倒计时（剩余时长），时间到自动提交 |
| 题目集合 | 每次独立随机 | 同卷，所有参赛者相同题目 |
| 是否扣次数 | 扣减答题次数 | 不扣减答题次数 |
| 退出处理 | 不保存记录 | 弹出强提示："退出后视为放弃本阶段答题，成绩以已作答部分计算" |

**顶部信息栏：**
- 赛事名称 + 当前阶段名称
- 题目进度（当前题 / 总题数）
- 倒计时（醒目展示，剩余 60 秒时变色提示）

**操作逻辑：**
- 倒计时归零时自动提交已作答内容
- 未作答题目默认为空（不计分）

---

### 6.5 赛事成绩与结果页

**页面定位：** 用户完成赛事答题后，以及赛事结果发布后的查看页面。

**答题刚完成时（结果待计算中）：**
- 展示"成绩计算中，请稍后查看"
- 提供返回赛事详情的入口

**成绩计算完成后：**

**个人赛成绩区：**
- 本阶段得分
- 本阶段排名（如 第 23 名 / 共 1200 人）
- 晋级状态（已晋级 / 未晋级 / 结果待公布）

**团体赛成绩区（团体赛补充展示）：**
- 个人成绩
- 团队总成绩（待团体赛成绩汇总算法确认后补充）
- 团队排名

**结果公布后：**
- 展示官方发布的晋级名单摘要
- 展示本人是否在晋级名单中

---

### 6.6 赛事排行榜页

**页面定位：** 展示当前赛事阶段的成绩排行。

**支持切换：**
- 个人排行 / 团队排行（团体赛时展示团队排行 Tab）

**个人排行列表字段：**
- 排名
- 姓名（脱敏）
- 所属单位
- 本阶段得分

**我的位置：**
- 固定展示本人排名行

**边界条件：**
- 阶段未开始或成绩未公布时：展示"排行榜将在阶段结束后公布"

---

## 7. 我的模块

### 7.1 我的主页

**页面定位：** 个人信息、关键数据和功能入口的汇总页。

**功能区块：**

**个人信息区：**
- 姓名
- 所属省份
- 所属单位
- 信息来源说明："信息来自国家级平台，如需修改请在国家级平台操作"

**次数账户摘要：**
- 基础次数余量
- 奖励次数余量
- 「查看明细」入口
- 「去获取次数」快捷入口（跳转学习联动引导页）

**快捷入口列表：**
- 参赛历史
- 练习历史（跳转至练习 Tab 的历史页）
- 学习联动记录

---

### 7.2 次数明细页

**页面定位：** 展示用户答题次数的来源和消耗明细。

**页面结构：**

**顶部汇总：**
- 基础次数余量
- 奖励次数余量
- 总可用次数

**明细列表（按时间倒序）：**

| 字段 | 说明 |
|------|------|
| 变动类型 | 消耗（练习赛）/ 获得（每日基础发放）/ 获得（学习任务奖励） |
| 变动次数 | +N 或 -N |
| 变动时间 | 时间戳 |
| 来源说明 | 如"完成指定课程奖励"/"练习赛消耗"/"每日基础次数" |
| 有效期 | 奖励次数的到期时间（当日有效，次日清零） |

**边界条件：**
- 无明细记录时展示空状态提示

---

### 7.3 学习联动记录页

**页面定位：** 展示用户在国家级平台完成学习任务的记录及对应次数到账情况。

**列表项字段：**
- 任务名称
- 完成时间
- 获得次数
- 到账状态（已到账 / 待到账 / 补偿中）

**待到账状态说明：**
- 展示说明文案："次数同步中，通常在 X 分钟内到账"
- 提供「刷新」按钮，手动触发一次查询

**边界条件：**
- 无学习联动记录时展示空状态提示

---

### 7.4 参赛历史页

**页面定位：** 展示用户参与过的所有正式赛事记录。

**列表项字段：**
- 赛事名称
- 参与阶段
- 参与时间
- 本人成绩
- 本人排名
- 晋级状态

**点击进入：**
- 展示该次赛事的完整成绩详情（复用 6.5 的赛事成绩结果页）

**边界条件：**
- 无参赛记录时展示空状态提示

---

## 8. 全局功能

### 8.1 通知中心

- 汇总管理员发送的赛事通知、晋级通知
- 列表展示（时间 + 标题）
- 点击查看详情

### 8.2 网络异常全局处理

- 网络不可用时展示全局 Toast 提示
- 不影响已加载内容的展示

### 8.3 加载状态

- 所有数据请求未返回时展示加载动画
- 超时时展示"加载失败，点击重试"
