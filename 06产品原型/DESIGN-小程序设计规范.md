# 灾害信息员竞赛SaaS平台 · 小程序端设计规范

## 1. 视觉主题与氛围

灾害信息员竞赛SaaS平台小程序面向基层灾害信息员学员，视觉风格以「专业、可信、轻量、激励」为核心。主色调采用应急科技蓝（`#1677FF`），传递应急管理领域的专业性与科技感；辅以活力橙（`#FF8F1F`）作为激励与竞赛情绪的色彩出口。整体界面以白色为底、蓝色为引、橙色点睛，营造清晰可信赖的竞赛氛围。

界面采用大量圆角卡片（`16px`）承载内容区块，降低阅读压力；功能入口使用图标+文字的组合形式，贴合移动端碎片化操作习惯。竞赛相关场景（排名、晋级、积分）适度使用勋章、徽章、进度条等游戏化视觉元素，增强参与感。

**核心特征：**
- 应急科技蓝（`#1677FF`）为品牌主色，贯穿导航、按钮、状态指示
- 活力橙（`#FF8F1F`）用于积分、参赛、激励等情绪高点
- 白色卡片 + 浅灰背景的双层表面结构
- `16px` 大圆角卡片为主内容容器，`8px` 小圆角用于内部元素
- 图标风格：线性+面性结合，竞赛类使用面性彩色图标增强识别

## 4. 组件样式

### 按钮

**主按钮（蓝色实心）**
- 背景：`#1677FF`
- 文字：`#FFFFFF`，`16px`，`font-weight: 600`
- 圆角：`8px`
- 高度：`48px`
- 禁用态：背景 `#A0C5FF`，文字 `#FFFFFF`
- 按压态：背景 `#0958D9`

**橙色激励按钮（积分/参赛）**
- 背景：`linear-gradient(135deg, #FF8F1F, #FFBA3B)`
- 文字：`#FFFFFF`，`16px`
- 圆角：`24px`（胶囊形）
- 高度：`40px`
- 带图标：左侧金币/奖杯图标 `20px`

**次级按钮（白色描边）**
- 背景：`#FFFFFF`
- 边框：`1px solid #E5E6EB`
- 文字：`#4E5969`
- 圆角：`8px`
- 高度：`40px`

**选项按钮（答题专用）**
- 默认：背景 `#F2F3F5`，文字 `#1D2129`，圆角 `8px`
- 选中：背景 `#E8F4FF`，边框 `2px solid #1677FF`，文字 `#1677FF`
- 正确：背景 `#E8FFF3`，边框 `2px solid #00B578`，文字 `#00B578`，带对勾图标
- 错误：背景 `#FFF2F0`，边框 `2px solid #F53F3F`，文字 `#F53F3F`，带叉号图标

### 卡片

**首页功能卡片**
- 背景：`#FFFFFF`
- 圆角：`16px`
- 内边距：`16px`
- 阴影：`0 2px 8px rgba(0, 0, 0, 0.04)`
- 图标区域：`48px × 48px`，圆角 `12px`，渐变背景

**竞赛阶段卡片**
- 背景：各阶段专属色 `8%` 透明度（如 `rgba(22, 119, 255, 0.08)`）
- 左侧边框：`4px` 实线，对应阶段色
- 圆角：`12px`
- 内边距：`16px`
- 状态标签：右上角圆角标签，"已结束"灰底，"进行中"蓝底，"未开始"橙底

**课程/内容卡片**
- 横向布局：左侧封面图 `120px × 80px`，圆角 `8px`
- 右侧标题 + 描述 + 标签
- 封面图可带渐变蒙层显示分类标签

### 导航

**顶部导航栏**
- 背景：`#FFFFFF`
- 标题：`#1D2129`，`18px`，居中
- 左侧返回箭头：`24px`，`#1D2129`
- 右侧操作：文字或图标，`#1677FF`
- 底部边框：`0.5px solid #E5E6EB`

**底部标签栏**
- 背景：`#FFFFFF`
- 高度：`56px + env(safe-area-inset-bottom)`
- 顶部边框：`0.5px solid #E5E6EB`
- 图标：`24px`，未选中 `#86909C`，选中 `#1677FF`
- 标签文字：`11px`，未选中 `#86909C`，选中 `#1677FF`
- 标签页：首页 / 练习 / 赛事 / 我的（按产品定义调整）

### 输入与搜索

**搜索框**
- 背景：`#F2F3F5`
- 圆角：`8px`
- 高度：`40px`
- 左侧搜索图标：`20px`，`#86909C`
- 占位文字：`#86909C`，`14px`
- 聚焦态：背景 `#FFFFFF`，边框 `1px solid #1677FF`

## 5. 布局原则

### 间距体系

基础单位为 `4px`，按以下倍数扩展：

| Token | 值 | 用途 |
|-------|-----|------|
| `space-xs` | `4px` | 图标与文字间距、紧凑内联间距 |
| `space-sm` | `8px` | 标签间距、小元素间隙 |
| `space-md` | `12px` | 卡片内部小间距、列表项间距 |
| `space-lg` | `16px` | 卡片内边距、模块间距 |
| `space-xl` | `20px` | 区块间距 |
| `space-2xl` | `24px` | 页面边距、大模块间距 |
| `space-3xl` | `32px` | 页面顶部安全区、大区块分隔 |

### 页面边距

- 水平页面边距：`16px`
- 卡片列表间距：`12px`
- 顶部状态栏适配：`44px`（iOS）/ `24px`（Android）

### 圆角层级

| 层级 | 值 | 用途 |
|------|-----|------|
| 小圆角 | `4px` | 标签、小按钮、输入框 |
| 中圆角 | `8px` | 按钮、封面图、选项卡片 |
| 大圆角 | `12px` | 功能卡片、竞赛卡片 |
| 超大圆角 | `16px` | 首页内容卡片、底部弹窗 |
| 全圆角 | `24px` / `999px` | 胶囊按钮、积分入口 |

## 6. 层级与阴影

小程序端阴影系统保持克制，深度主要通过背景色对比实现：

| 层级 | 阴影值 | 用途 |
|------|--------|------|
| 表面阴影 | `0 2px 8px rgba(0, 0, 0, 0.04)` | 卡片悬浮感 |
| 抬升阴影 | `0 4px 16px rgba(0, 0, 0, 0.08)` | 底部弹窗、浮层 |
| 强调阴影 | `0 0 0 3px rgba(22, 119, 255, 0.15)` | 按钮聚焦态、选中态外发光 |

## 8. 竞赛场景专用规范

### 答题场景

- 题目区域：白色卡片，`16px` 圆角，`16px` 内边距
- 题型标签：左侧竖线 `4px` + 文字，如"单选题"蓝线，"多选题"橙线
- 进度指示：顶部进度条，`4px` 高，`#1677FF` 已答，`#E5E6EB` 未答
- 题号：右上角 `14px`，如 "1/5"
- 选项间距：`12px`
- 提交按钮：吸底，主按钮样式

### 排名与晋级

- 排名列表：前三名使用特殊样式（冠/亚/季军徽章 + 金色/银色/铜色元素）
- 晋级状态：使用阶段专属色标签 + 箭头图标
- 分数展示：`36px` 大字号，`#1677FF` 或 `#FF8F1F`
- 正确率环：圆环进度条，`#1677FF` 填充，`#E5E6EB` 背景

### 赛事阶段展示

- 时间轴布局：垂直排列，左侧阶段图标 + 右侧信息卡片
- 进行中状态：图标带脉冲动画或高亮边框
- 已结束状态：整体置灰（透明度 `60%`）

## 9. Do's & Don'ts

### Do

- 使用科技蓝（`#1677FF`）作为所有核心交互和竞赛状态的主色
- 首页功能入口统一使用 `48px` 彩色面性图标 + 文字说明
- 答题选项使用充足的触控区域（最小 `48px` 高度）
- 竞赛相关数字（积分、排名、分数）使用大字号强调
- 保持页面边距一致（`16px`）
- 使用阶段专属色区分练习赛/初赛/复赛/决赛

### Don't

- 不要引入除蓝、橙、绿、红之外的额外品牌色
- 不要使用直角卡片（最小 `8px` 圆角）
- 不要在同一页面使用超过两种渐变背景
- 不要对非竞赛相关元素使用橙色，避免稀释激励感
- 不要在答题界面使用过多装饰元素干扰阅读
- 避免文字过小（正文不低于 `14px`）

## 10. 响应式适配

小程序端以微信小程序标准视口为基准：

- 设计稿基准：`375px` 宽度（iPhone 标准）
- 安全区适配：底部 `env(safe-area-inset-bottom)`，顶部导航栏高度 `44px`
- 大屏适配（`> 414px`）：卡片内边距增至 `20px`，图标增大 `20%`
- 小屏适配（`< 360px`）：功能图标缩小至 `40px`，间距缩减 `20%`
