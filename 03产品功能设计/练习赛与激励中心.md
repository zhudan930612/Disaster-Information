## 模块概述

### 业务场景

练习赛与激励中心承担正式赛事之外的常态化练习能力，是用户日常提升业务技能、积累答题经验和熟悉题型的主要入口。用户在小程序`练习`页中查看当日剩余练习次数、基础次数、奖励次数，并通过`开始练习`进入练习答题流程；当日次数用尽后，系统引导用户前往国家培训平台完成课程学习或累计学习时长，从而获得新的奖励练习次数。后台则通过`配置 > 练习规则配置`和`配置 > 学习联动规则配置`两类配置页维护次数规则和学习联动发放规则。

**核心价值**：
- **参练用户**：即使没有正式赛事，也可以持续练习、查看错题和历史练习表现
- **平台运营/省级运营**：可以通过配置而不是改代码的方式维护基础练习次数和奖励规则
- **整个产品闭环**：把国家培训平台的“学习”行为转成竞赛平台的“练习”激励，形成学练结合闭环

**主要功能**：
1. **练习首页展示**：展示今日剩余练习次数、基础次数、奖励次数和练习入口
2. **练习结果沉淀**：沉淀错题本和练习记录，支持用户复盘
3. **练习次数明细**：查看次数来源、消耗和剩余情况
4. **学习联动发放**：按课程数或学习时长发放奖励练习次数
5. **后台规则配置**：配置基础练习次数、重置规则、奖励叠加规则和学习联动规则

**适用范围**：
- 小程序`练习`一级页面
- 小程序练习页下的`错题本`、`练习记录`、`练习次数明细`
- 后台`配置 > 练习规则配置`
- 后台`配置 > 学习联动规则配置`

### 受众角色

#### 1. 参练用户（小程序端）
**角色定义**：已登录并已同步到竞赛平台的灾害信息员用户，可在日常进入练习页参加练习。

**使用场景**：
- 查看今天还能练习多少次
- 使用剩余次数进入练习
- 查看错题和练习记录
- 次数不足时去国家培训平台学习以获取更多练习次数

**典型用户**：
- 村级、乡级、县级、市级灾害信息员
- 还未进入正式赛事阶段，但需要先练习备赛的用户

**核心诉求**：
- 练习入口清晰
- 次数口径透明
- 用完后知道去哪里继续获得机会

#### 2. 平台运营/省级运营（后台运营端）
**角色定义**：负责维护练习次数规则和学习联动奖励规则的后台账号。

**使用场景**：
- 配置每日基础练习次数
- 配置奖励次数是否可叠加、有效期和消耗顺序
- 配置“完成 X 个课程”或“累计 X 分钟学习”后发放多少次练习次数
- 校验规则是否生效

**典型用户**：
- 平台超级管理员
- 省级 SaaS 运营管理员

**核心诉求**：
- 不改代码即可维护次数规则
- 奖励规则清晰可追踪
- 学习联动发放结果可校验

#### 3. 培训平台同步服务（系统角色）
**角色定义**：负责把国家培训平台中的学习行为结果同步到竞赛平台，用于触发奖励练习次数。

**功能职责**：
- 同步课程完成数量
- 同步累计学习时长
- 将满足条件的奖励结果写入练习次数账户

**说明**：本文只记录同步结果在竞赛平台中的使用方式，不展开接口协议和同步技术细节。

### 业务价值

**对参练用户**：
- 在正式比赛外持续提升能力
- 通过错题本和练习记录看到个人进步

**对运营管理方**：
- 用配置完成奖励机制维护
- 可以快速调整不同阶段的练习激励强度

**对整个平台**：
- 让国家培训平台和竞赛平台形成行为闭环
- 为正式赛事提供热身和能力预热通道

## 通用规则

### 1. 开放范围与访问规则
- 练习功能面对所有已同步到竞赛平台的用户开放，不受省份开通状态限制。
- 即使省份服务有效期到期，正式竞赛能力关闭，练习功能仍然保留可用。
- 未从国家培训平台同步到竞赛平台的用户，不出现在练习相关统计、记录和次数计算范围内。

### 2. 练习次数口径规则
- `今日剩余练习次数 = 基础剩余次数 + 奖励剩余次数`
- `基础剩余次数 = 当日配置基础次数 - 当日已消耗基础次数`
- `奖励剩余次数 = 当前仍在有效期内的奖励次数总和 - 已消耗奖励次数`
- 基础练习次数按配置规则重置，默认按自然日重置。
- 奖励次数是否允许叠加、有效期多久、先消耗基础还是先消耗奖励，均通过后台配置控制。
- 如后台未做特殊调整，默认采用`基础次数优先消耗`。

### 3. 学习联动发放规则
- 学习联动奖励来源于国家培训平台同步过来的学习结果。
- 奖励触发方式支持两类：`完成课程数`、`累计学习时长（分钟）`。
- 每条联动规则至少包含：触发类型、阈值、奖励次数、生效状态。
- 奖励发放成功后，应立即更新练习页中的奖励次数和今日剩余次数。
- 如果同步尚未完成，即使用户已经在培训平台学习，竞赛平台也不会先行预估或展示未到账次数。

### 4. 练习结果沉淀规则
- 每完成一次练习，系统生成一条练习记录。
- 练习中答错的题目进入错题本。
- 同一题目重复答错时，在错题本中累计错误次数，而不是重复生成多条完全独立的错题卡片。
- 练习记录默认按时间倒序展示，最新记录排在最前。

### 5. 页面展示与交互规则
- 练习首页顶部展示今日剩余练习次数，并区分展示基础次数和奖励次数。
- 当今日剩余练习次数大于 0 时，`开始练习`按钮可点击。
- 当今日剩余练习次数等于 0 时，`开始练习`按钮置灰不可点击，并展示获取更多练习次数的引导文案。
- `错题本`与`练习记录`通过 Tab 切换展示。
- 列表无数据时统一展示空状态。

### 6. 模块边界规则
- 练习题源、练习赛规则和练习赛题库关联，来源于[赛事配置中心](./赛事配置中心.md)中的`练习赛`配置。
- 用户身份、组织归属和同步范围，来源于[用户与人员中心](./用户与人员中心.md)。
- 本文不展开正式答题页、交卷页和答题引擎细节，只记录练习入口、次数、激励和结果沉淀。

## 完整业务流程

### 流程概述

练习赛与激励中心涉及两个主要流程：参练用户的小程序练习流程，以及后台运营的规则维护流程。

#### 一、参练用户练习流程
参练用户的完整流程包括：
- 进入练习首页查看今日剩余练习次数
- 点击开始练习进入答题
- 完成练习后沉淀错题本和练习记录
- 次数不足时前往国家培训平台学习
- 学习结果同步后获取新的奖励练习次数

#### 二、后台配置流程
后台运营的完整流程包括：
- 在`练习规则配置`中维护基础练习次数和奖励次数通用规则
- 在`学习联动规则配置`中维护课程数/学习时长到奖励次数的映射关系
- 保存规则后生效
- 通过前台次数变化和记录明细校验规则结果

### 详细流程步骤

#### 步骤1：用户进入练习页
- 用户进入小程序底部一级导航`练习`
- 页面顶部展示今日剩余练习次数和练习次数明细入口
- 页面中部展示基础次数、奖励次数和`开始练习`按钮

#### 步骤2：判断次数是否足够
- 若今日剩余练习次数大于 0，用户可点击`开始练习`
- 若今日剩余练习次数等于 0，页面显示次数已用尽引导文案和`获取练习次数`入口

#### 步骤3：完成练习并沉淀数据
- 用户进入练习答题流程
- 系统按练习赛配置的题库和规则出题
- 练习结束后，生成练习记录
- 答错题目进入错题本

#### 步骤4：学习联动发放奖励次数
- 用户从练习页跳转至国家培训平台
- 完成课程数或累计学习时长达到联动规则阈值
- 国家培训平台同步学习结果
- 竞赛平台按生效规则发放奖励练习次数

#### 步骤5：后台维护规则
- 运营在后台维护基础练习次数和学习联动规则
- 保存后规则立即生效于当前租户/省份范围内的用户

## 参练用户操作流程

## 查看练习首页

### 页面价值
- 让用户一眼看到今天还能练习多少次
- 给出开始练习和获取更多练习次数的主入口

### 页面结构
- 顶部信息卡
- `练习次数明细`入口
- `基础次数`卡片
- `奖励次数`卡片
- `开始练习`按钮
- 下方 Tab：`错题本`、`练习记录`
- 底部一级导航：`练习`、`竞赛`、`我的`

### 当前已确认展示字段
- `今日剩余练习次数`
- `基础次数`
- `奖励次数`
- `练习次数明细`
- `开始练习`
- `错题本`
- `练习记录`

### 业务规则
- 今日剩余练习次数用大号数字展示，是页面首要信息。
- `基础次数`和`奖励次数`分别展示剩余值，而不是累计历史值。
- 页面默认进入`错题本`Tab。
- 当剩余次数为 0 时，按钮置灰，并展示橙色提示文案。

## 开始练习

### 页面价值
- 提供正式进入练习答题的入口

### 当前已确认操作
- 当今日剩余练习次数大于 0 时，点击`开始练习`
- 当今日剩余练习次数等于 0 时，按钮不可点击

### 业务规则
- 进入练习时，系统按[赛事配置中心](./赛事配置中心.md)中配置的`练习赛`规则和关联题库出题。
- 每次进入练习视为一次练习机会消耗。
- 默认先消耗基础次数，再消耗奖励次数；如后台修改消耗顺序，则按配置执行。
- 练习答题流程本身不在本文展开。

## 查看练习次数明细

### 页面价值
- 让用户知道次数从哪里来、被怎么用掉

### 入口
- 练习首页顶部卡片右上角`练习次数明细`

### 建议页面结构
- 顶部展示当前剩余基础次数和奖励次数
- 下方列表展示次数流水

### 建议列表字段
- `时间`
- `类型`：基础发放 / 奖励发放 / 次数消耗 / 次数过期
- `变动值`
- `变动后剩余次数`
- `来源说明`

### 业务规则
- 基础次数发放通常在每日重置时生成一条发放流水。
- 奖励次数发放在学习联动触发成功后生成流水。
- 用户每完成一次练习后生成一次消耗流水。
- 如奖励次数存在有效期，到期后生成过期流水。

## 获取更多练习次数

### 页面价值
- 在次数用尽时，明确引导用户通过学习继续获取练习机会

### 当前已确认展示
- 当次数用尽时，页面文案提示：
  - 今日练习次数已用尽
  - 前往国家培训平台学习课程，即可获得更多练习机会
- 页面提供`获取练习次数 >`入口

### 业务规则
- 点击`获取练习次数`后跳转国家培训平台。
- 跳转前保持当前登录态；如登录态失效，按统一登录流程处理。
- 用户回到竞赛平台后，系统自动刷新奖励次数。
- 如学习结果尚未同步到账，页面继续显示当前剩余次数，不展示预发放次数。

- `最近错误时间`
- `累计错误次数`

- 右侧展示错误次数增量，如`+1`

- 错题本默认按最近错误时间倒序排列。
- 列表无数据时展示空状态。
- 默认支持点击列表项查看题目详情、正确答案和答案解析。

## 后台练习规则配置流程

## 查看练习规则配置页面

### 入口路径
- 后台 `配置 > 练习规则配置`

### 页面价值
- 统一维护当前租户/省份范围内的练习次数基础规则

### 建议页面结构
- 基础次数配置区
- 重置规则配置区
- 奖励次数通用规则区
- 保存操作区

### 建议字段
- `每日基础练习次数`
- `重置方式`
- `重置时间`
- `次数消耗顺序`
- `奖励次数是否允许叠加`
- `奖励次数有效期类型`
- `奖励次数有效期值`

### 业务规则
- 配置结果对当前租户/省份范围内用户生效。
- 如未做特殊配置，默认按自然日 00:00 重置。
- 页面底部提供`取消`、`保存`按钮。

## 配置基础练习次数规则

### 页面价值
- 控制每个用户每天天然可练习多少次

### 当前建议字段说明
- `每日基础练习次数`：整数，必须大于等于 0
- `重置方式`：默认`自然日`
- `重置时间`：默认`00:00`

### 业务规则
- 每日基础练习次数对所有已同步用户生效。
- 当值为 0 时，用户仅能依赖奖励次数练习。
- 基础次数重置时，仅重置基础账户，不影响尚未到期的奖励次数。

## 配置奖励次数叠加与有效期

### 页面价值
- 控制学习联动发放的奖励次数如何使用

### 当前建议字段说明
- `奖励次数是否允许叠加`：开关
- `奖励次数有效期类型`：当日有效 / N 日有效 / 永久有效
- `奖励次数有效期值`：当选择 N 日有效时填写整数
- `次数消耗顺序`：基础优先 / 奖励优先

### 业务规则
- 如允许叠加，则多次触发奖励规则可累计增加奖励次数。
- 如不允许叠加，则用户在已有未用完奖励次数时，不重复累计相同规则发放的次数。
- 到期未使用的奖励次数自动失效，并记录失效流水。

## 保存练习规则

### 页面价值
- 让配置结果正式生效

### 建议交互
- 点击`保存`后校验必填项
- 校验通过则保存成功
- 校验失败则在对应字段下提示错误

### 常规校验规则
- `每日基础练习次数`不能为空
- `每日基础练习次数`必须为非负整数
- 选择`N 日有效`时，必须填写有效期值
- `重置时间`需为有效时间格式

### 默认反馈方式
- 保存成功提示：`保存成功`
- 保存后留在当前页，并刷新展示最新配置

## 后台学习联动规则配置流程

## 查看学习联动规则配置页面

### 入口路径
- 后台 `配置 > 学习联动规则配置`

### 页面价值
- 维护“学习行为 -> 奖励练习次数”的映射关系

### 建议页面结构
- 规则列表区
- 查询筛选区
- 新建规则按钮
- 启停/编辑操作区

### 建议列表字段
- `规则名称`
- `触发类型`
- `触发阈值`
- `奖励练习次数`
- `是否可重复触发`
- `生效状态`
- `更新时间`
- `操作`

### 业务规则
- 支持存在多条联动规则。
- 规则默认按启用状态参与发放计算。
- 同一时间允许同时存在“按课程数触发”和“按学习时长触发”的规则。

## 新建学习联动规则

### 页面价值
- 新增一条学习联动发放规则

### 建议字段
- `规则名称`
- `触发类型`：完成课程数 / 累计学习时长
- `触发阈值`
- `奖励练习次数`
- `是否可重复触发`
- `生效状态`
- `规则说明`

### 业务规则
- 当`触发类型 = 完成课程数`时，阈值单位为“个课程”。
- 当`触发类型 = 累计学习时长`时，阈值单位为“分钟”。
- `奖励练习次数`必须为正整数。
- 如开启`是否可重复触发`，则用户每达到一次阈值可再次获得奖励。
- 如关闭`是否可重复触发`，则同一用户仅首次达标时发放一次。

## 编辑或启停联动规则

### 页面价值
- 对已有规则进行调整或临时停用

### 建议操作
- `编辑`
- `启用`
- `停用`
- `删除`

### 业务规则
- 停用后的规则不再参与后续奖励发放，但历史已发放次数不回滚。
- 编辑后的规则仅影响编辑生效后的新同步结果。
- 删除规则前应弹出二次确认。

### 默认反馈方式
- 启用成功提示：`启用成功`
- 停用成功提示：`停用成功`
- 编辑成功提示：`保存成功`
- 删除成功提示：`删除成功`

## 校验奖励发放结果

### 页面价值
- 帮助运营确认学习联动规则是否真正生效

### 校验方式
- 在练习页查看奖励次数是否增加
- 在练习次数明细中查看是否生成奖励发放流水
- 在用户档案中查看累计练习次数是否随着后续练习增长

### 业务规则
- 学习结果同步成功但未达到阈值时，不发放奖励次数。
- 达到阈值但规则停用时，不发放奖励次数。
- 发放成功后，应能在练习次数明细中看到来源说明，如`学习联动奖励`

## 特殊场景

## 今日练习次数已用尽

### 场景说明
- 用户今日剩余练习次数为 0

### 页面表现
- `开始练习`按钮置灰不可点击
- 页面显示橙色引导文案
- 页面显示`获取练习次数 >`入口

### 业务规则
- 用户不能通过刷新页面重新获得次数。
- 只有基础次数自然日重置，或奖励次数发放成功后，页面才会重新可练习。

## 培训平台数据未同步

### 场景说明
- 用户已在国家培训平台学习，但学习结果尚未同步到竞赛平台

### 页面表现
- 练习页仍显示当前已到账的基础次数和奖励次数
- 不预估、不透出“待到账”次数

### 业务规则
- 未同步过来的学习结果不纳入当前次数计算。
- 未同步到竞赛平台的用户和学习结果，不计入练习相关统计。

## 服务到期后仅保留练习

### 场景说明
- 当前省份/租户服务有效期到期

### 页面表现
- 正式竞赛功能关闭
- 练习功能仍保留

### 业务规则
- 练习页继续展示并允许用户使用剩余次数练习。
- 后台练习规则和学习联动规则仍可按当前项目约定继续运行。

## 与赛事配置中心的边界

### 场景说明
- 练习赛与正式赛事都涉及题库和答题规则，容易混淆

### 边界规则
- `练习赛与激励中心`负责：
  - 练习首页
  - 次数口径
  - 奖励发放
  - 错题本与练习记录
  - 后台练习次数与学习联动规则配置
- `赛事配置中心`负责：
  - 练习赛题库来源
  - 练习赛题量、限时、赛制规则
  - 练习赛是否开放及关联题库
