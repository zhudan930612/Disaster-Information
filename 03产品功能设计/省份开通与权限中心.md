## 模块概述

### 业务场景

省份开通与权限中心是省级 SaaS 产品化的总开关模块，负责决定“哪个省可以用正式竞赛功能”“可以用到什么时候”“能用哪些赛段和赛制”。平台运营超级管理员在平台级后台为省份开通服务、设置有效期，并决定初赛、复赛、决赛下个人赛和团队赛的可用范围；用户访问前台正式竞赛页、后台省级赛事配置页和正式竞赛执行页时，系统都要先经过本模块的校验。练习功能不受本模块限制。

**核心价值**：
- **平台运营**：可以统一管理多省交付状态，而不是人工线下维护
- **省级管理员**：明确知道本省当前能配置和能运营的正式竞赛范围
- **系统治理**：通过统一规则拦截未开通、省份不匹配或已到期场景，避免越权和误开放

**主要功能**：
1. **省份开通管理**：按省配置开通/关闭状态
2. **服务有效期管理**：按省配置服务开始和结束时间
3. **赛段/赛制范围管理**：按省控制初赛/复赛/决赛下个人赛、团队赛是否可用
4. **正式竞赛入口拦截**：控制前台正式竞赛访问
5. **后台权限联动**：联动省级赛事配置、报名、结果等正式竞赛可操作范围

**适用范围**：
- 平台运营后台中的省份开通管理页面
- 前台正式竞赛访问校验
- 后台正式竞赛相关菜单和页面的权限联动

### 受众角色

#### 1. 平台运营超级管理员（平台运营后台）
**角色定义**：平台级统一管理者，负责开通省份、维护服务状态和控制正式竞赛可用范围。

**使用场景**：
- 为签约省份开通服务
- 设置省份服务有效期
- 控制初赛/复赛/决赛下的个人赛和团队赛是否可用
- 排查某省为何无法进入正式竞赛

**典型用户**：
- 平台运营超级管理员
- 平台级交付管理人员

**核心诉求**：
- 一眼看到各省当前服务状态
- 不依赖开发即可调整开关范围
- 对到期和未开通情况有清晰控制

#### 2. 省级管理员（省级后台）
**角色定义**：本省赛事运营方，只能查看本省的服务状态，并在被授予范围内配置和运营正式赛事。

**使用场景**：
- 查看本省是否已开通正式竞赛能力
- 查看本省目前能配置哪些赛段和赛制
- 在权限允许范围内配置赛事和执行报名

**典型用户**：
- 省级赛事管理员
- 省级运营人员

**核心诉求**：
- 看到本省当前开通范围
- 被限制的功能不要误显示或误允许操作

#### 3. 参赛用户（小程序端）
**角色定义**：访问前台正式竞赛和练习功能的终端用户。

**使用场景**：
- 进入正式竞赛页查看是否有可参与赛事
- 服务未开通或到期时获得明确反馈
- 在任何情况下继续使用练习功能

**核心诉求**：
- 正式竞赛是否可用要明确
- 不能参与时原因要清楚
- 练习入口始终稳定

#### 4. 系统权限校验服务（系统角色）
**角色定义**：在前台、后台访问正式竞赛能力时执行统一校验的系统能力。

**功能职责**：
- 校验用户所属省份
- 校验省份开通状态
- 校验服务有效期
- 校验赛段/赛制是否被授权可用

**说明**：本文只记录业务校验顺序和口径，不展开接口与中间件实现细节。

### 业务价值

**对平台运营**：
- 把“一省一赛”的交付控制收归平台统一管理
- 避免多个省份交付状态靠口头同步

**对省级管理员**：
- 让可配置范围与可运营范围保持一致
- 减少“页面看得到但实际上不能用”的误操作

**对整个系统**：
- 实现正式竞赛权限和服务状态的统一拦截
- 保持练习开放与正式竞赛管控并存

## 通用规则

### 1. 省份开通规则
- 省份开通只按省级维度控制，不区分国家级观察场景的单独开通口径。
- 省份开通控制的是`正式竞赛相关能力`，不是练习功能总开关。
- 同一时间内，一个省只能有一个当前生效的服务状态配置。

### 2. 服务有效期规则
- 每个已开通省份都应配置服务开始日期和服务结束日期。
- 当当前时间早于服务开始时间时，视为未生效。
- 当当前时间晚于服务结束时间时，视为已到期。
- 服务有效期到期后，正式竞赛功能关闭，仅保留练习功能可用。

### 3. 赛段与赛制规则
- 可用赛段包含：`初赛`、`复赛`、`决赛`。
- 每个赛段下可独立控制：`个人赛`、`团队赛`。
- 同一赛段允许个人赛和团队赛同时开启。
- 平台级开通范围只定义“这个省可以用哪些赛段/赛制”，不替代该省具体赛事规则配置。

### 4. 权限联动规则
- 平台运营超级管理员可查看和编辑所有省份开通配置。
- 省级管理员只能查看本省服务状态和可用范围，不能编辑跨省开通配置。
- 被关闭或未授予的赛段/赛制，在省级后台应隐藏或置灰，避免误配置。

### 5. 访问拦截规则
- 前台正式竞赛访问前，系统按“用户所属省份 -> 省份开通状态 -> 服务有效期 -> 赛段/赛制开通范围 -> 赛事发布状态”的顺序校验。
- 后台正式竞赛配置和执行页面访问前，系统按“操作账号角色 -> 所属省份 -> 省份开通状态 -> 服务有效期 -> 赛段/赛制授权范围”的顺序校验。
- 练习功能完全不受省份开通限制。

### 6. 与系统管理底座的关系
- 省份主数据默认依托现有机构管理中的省级机构节点。
- 账号、角色、导航和菜单裁剪依托现有系统管理底座实现。
- 本模块不负责维护机构树本身，只负责把机构树中的省份节点转化为“服务开通对象”。

## 完整业务流程

### 流程概述

本模块包含两个主要流程：平台运营开通省份流程，以及系统访问校验与权限联动流程。

#### 一、平台运营开通省份流程
平台运营超级管理员的完整流程包括：
- 查看省份开通列表
- 选择某省配置开通状态
- 设置服务有效期
- 配置初赛/复赛/决赛下个人赛、团队赛可用范围
- 保存并生效

#### 二、系统访问校验流程
系统在用户访问和后台操作时的完整流程包括：
- 识别访问主体所属省份与角色
- 校验省份是否开通且是否在有效期内
- 校验赛段/赛制是否被授权
- 校验通过后放行，否则拦截正式竞赛能力

### 详细流程步骤

#### 步骤1：平台运营进入省份开通页面
- 平台运营超级管理员进入平台运营后台
- 打开`省份开通管理`页面
- 查看各省当前服务状态

#### 步骤2：配置某省开通状态
- 选择目标省份
- 设置该省是否开通正式竞赛服务
- 如开通，则继续配置有效期和赛段/赛制范围

#### 步骤3：保存并生效
- 系统校验日期和开通范围
- 校验通过后保存
- 新规则立即影响前台正式竞赛入口和后台正式竞赛操作范围

#### 步骤4：访问正式竞赛能力时执行校验
- 用户或管理员访问正式竞赛相关页面
- 系统执行统一校验
- 校验通过则放行，否则拦截

#### 步骤5：服务到期后自动切换状态
- 当服务结束日期已过
- 系统自动将该省视为已到期状态
- 正式竞赛相关能力关闭，练习功能继续可用

## 平台运营操作流程

## 查看省份开通列表

### 页面价值
- 让平台运营统一查看所有省份当前服务状态

### 入口路径
- 平台运营后台 -> 省份开通管理

### 建议页面结构
- 查询筛选区
- 省份开通列表区
- 行内配置入口

### 建议列表字段
- `省份名称`
- `开通状态`
- `服务开始日期`
- `服务结束日期`
- `当前服务状态`
- `可用正式竞赛范围`
- `最后更新时间`
- `更新人`
- `操作`

### 业务规则
- `当前服务状态`建议分为：未开通 / 未生效 / 服务中 / 已到期。
- `可用正式竞赛范围`建议以摘要方式展示，例如：`初赛-个人/团队；复赛-个人；决赛-关闭`。
- `操作`列默认包含：`查看`、`配置`。

## 按条件查询省份开通状态

### 页面价值
- 快速定位某个省当前是否开通、是否到期

### 建议查询字段
- `省份名称`
- `开通状态`
- `当前服务状态`

### 建议操作
- `查询`
- `重置`

### 业务规则
- 省份名称支持模糊查询。
- 开通状态支持：开通 / 关闭。
- 当前服务状态支持：未开通 / 未生效 / 服务中 / 已到期。

## 配置省份开通状态

### 页面价值
- 统一控制某省正式竞赛服务总开关

### 建议页面结构
- 基础信息区
- 服务状态配置区
- 保存操作区

### 建议字段
- `省份名称`：只读
- `开通状态`：开通 / 关闭
- `配置说明`：选填

### 业务规则
- 关闭状态下，该省正式竞赛相关能力一律不可用。
- 关闭状态不影响练习功能。
- 关闭状态下，允许保留历史配置记录，但不对前台和省级正式竞赛操作生效。

## 配置服务有效期

### 页面价值
- 明确一个省的服务可用时段

### 建议字段
- `服务开始日期`
- `服务结束日期`

### 常规校验规则
- 当`开通状态 = 开通`时，开始日期和结束日期必填。
- 开始日期不能晚于结束日期。
- 结束日期不能为空。

### 业务规则
- 服务未生效时，前后台正式竞赛能力都不开放。
- 服务到期后，自动关闭正式竞赛能力。
- 如需继续使用，平台运营重新调整服务结束日期即可恢复。

## 配置可用赛段与赛制范围

### 页面价值
- 控制该省可以使用哪些正式竞赛能力

### 建议页面结构
- 赛段/赛制矩阵配置区

### 建议配置项
- `初赛 - 个人赛`
- `初赛 - 团队赛`
- `复赛 - 个人赛`
- `复赛 - 团队赛`
- `决赛 - 个人赛`
- `决赛 - 团队赛`

### 业务规则
- 每个开关独立控制。
- 同一赛段下，个人赛和团队赛可同时开启。
- 全部关闭时，表示该省当前仅保留练习和通用能力，不开放任何正式竞赛赛段/赛制。
- 赛段/赛制范围只定义“可不可以用”，不定义具体比赛规则；具体时间、题库、答题次数等仍在赛事配置中心维护。

## 保存并生效省份开通规则

### 页面价值
- 让平台运营配置完成后的规则即时生效

### 建议交互
- 页面底部提供`取消`、`保存`按钮
- 点击保存后执行字段校验
- 校验通过后保存成功并刷新列表

### 默认反馈方式
- 保存成功提示：`保存成功`
- 校验失败时，在对应字段下方提示错误

### 常规校验规则
- 开通状态为开通时，服务有效期必填
- 服务结束时间不得早于开始时间
- 至少允许为某省保存“仅练习、无正式竞赛”的状态，因此赛段/赛制可用范围允许全不选

## 查看省份当前服务状态

### 页面价值
- 帮助运营确认当前配置是否真正生效

### 建议查看维度
- 当前是否开通
- 当前是否在有效期内
- 当前可用赛段/赛制范围
- 到期时间

### 业务规则
- 列表中的`当前服务状态`应实时根据当前时间计算，不依赖人工手动改状态。
- 服务状态变化后，前后台正式竞赛入口应同步受影响。

## 访问校验与权限联动流程

## 前台正式竞赛入口校验

### 页面价值
- 避免未开通或到期省份用户进入正式竞赛

### 校验顺序
1. 用户是否已同步到竞赛平台
2. 用户是否有所属省份
3. 所属省份是否为开通状态
4. 所属省份是否在服务有效期内
5. 访问的赛段/赛制是否在开通范围内
6. 对应赛事是否已在赛事配置中心发布

### 业务规则
- 任一校验不通过，都不放行正式竞赛能力。
- 校验失败时，前台应展示明确提示或空状态说明。
- 练习页不参与上述校验链路。

## 后台赛事配置权限校验

### 页面价值
- 确保省级管理员只能配置被授权的赛段和赛制

### 校验顺序
1. 当前账号是否为省级管理员
2. 当前账号所属省份是否与被配置省份一致
3. 所属省份是否开通且在有效期内
4. 当前操作赛段/赛制是否在平台开通范围内

### 业务规则
- 未授予的赛段/赛制在省级后台中应隐藏或置灰。
- 已授予的赛段/赛制才允许在赛事配置中心进入编辑。
- 平台运营超级管理员不受省份单一视角限制，可进行平台级维护。

## 后台正式竞赛操作权限校验

### 页面价值
- 控制报名入口、竞赛结果、人员详情等正式竞赛操作入口

### 涉及页面
- `赛事配置中心`
- `竞赛执行中心`
- `统计分析与管理驾驶舱`中的正式竞赛数据

### 业务规则
- 当省份未开通或已到期时，省级管理员不应继续操作正式竞赛配置、报名和结果管理。
- 平台级运营账号可保留运维视角和查看能力。
- 是否保留省级历史数据只读视图，可按后续菜单裁剪实现；本模块默认只控制“是否可操作正式竞赛能力”。

## 省级管理员查看本省开通状态

### 页面价值
- 让省级管理员明确知道自己当前能用哪些能力

### 建议展示内容
- 本省当前服务状态
- 服务有效期
- 可用赛段/赛制范围

### 业务规则
- 省级管理员默认只读查看本省，不可修改。
- 如本省未开通或已到期，可在页面提示联系平台运营。

## 特殊场景

## 未开通省份访问正式竞赛

### 场景说明
- 用户所属省份未被平台开通

### 页面表现
- 前台正式竞赛页不放行正式比赛能力
- 后台省级正式竞赛相关操作不可用

### 业务规则
- 练习功能继续可用。
- 省级管理员如登录后台，应只能看到被保留的通用功能和练习相关能力。

## 服务有效期到期

### 场景说明
- 当前时间已晚于服务结束日期

### 页面表现
- 正式竞赛入口关闭
- 省级正式竞赛操作关闭
- 练习功能保留

### 业务规则
- 到期状态按时间自动判断，无需人工手动切换。
- 平台运营修改服务结束日期后，规则可恢复生效。

## 仅开放部分赛段或赛制

### 场景说明
- 平台运营只给某省开放部分赛段或赛制，例如仅开放初赛个人赛

### 页面表现
- 省级后台只显示或仅允许配置被开放的赛段/赛制
- 前台只展示对应已发布且已授权的正式赛事内容

### 业务规则
- 未开放的赛段/赛制，即使赛事配置中心已有配置草稿，也不应对外开放。
- 同一阶段个人赛和团队赛的开关彼此独立。

## 练习功能不受省份开通限制

### 场景说明
- 无论某省是否开通正式竞赛，用户都可以使用练习

### 页面表现
- 小程序`练习`页始终可进入
- 练习次数、错题本、练习记录正常运行

### 业务规则
- 练习模块完全独立于省份开通状态。
- 即使服务到期，也仅关闭正式竞赛能力，不关闭练习能力。

## 与系统管理中心的边界

### 场景说明
- 本模块容易和机构管理、角色管理、账号管理混淆

### 边界规则
- `省份开通与权限中心`负责：
  - 省份服务开通状态
  - 服务有效期
  - 赛段/赛制可用范围
  - 正式竞赛前后台入口拦截与权限联动
- `系统管理中心`负责：
  - 机构树维护
  - 角色模板维护
  - 账号维护
  - 导航菜单配置
- 本模块默认复用系统管理底座，但不替代系统管理底座本身。
