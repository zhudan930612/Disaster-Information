## 2. 模块概述

题库中台是平台的内容底座，支撑练习赛和正式赛事的题目供给。采用**三层架构**：

| 层级 | 定位 | 管理方 | 使用场景 |
|------|------|--------|---------|
| 平台共用题库 | 全平台共享，支撑所有用户的练习赛 | 平台超管/运营 | 练习赛随机组题 |
| 省级自主题库 | 各省为正式赛事自主配置的题库 | 省级管理员 | 正式赛事同卷锁定 |
| 题目权限公开 | 平台题目的可见性控制 | 平台超管/运营 | 控制哪些题目可供各省选用 |

**核心规则**：
- 练习赛统一使用平台共用题库，各省不可干预
- 正式赛事各省可自主配置题库（从平台共用题库选用 + 加入自有题目）
- 平台题目支持权限公开，私有题目各省不可见不可选

## 4. 数据模型

### 4.1 题库（QuestionBank）

| 字段 | 类型 | 说明 |
|------|------|------|
| 题库ID | 主键 | 唯一标识 |
| 题库名称 | 文本 | 如"2025练习赛题库"、"浙江省初赛题库" |
| 题库类型 | 枚举 | platform（平台共用）/ province（省级自主） |
| 所属租户ID | 外键 | 平台共用题库为空，省级题库关联对应省级机构ID |
| 题目数量 | 数字 | 当前题库下的题目总数 |
| 创建人 | 外键 | 创建者账号 |
| 创建时间 | 时间 | — |
| 更新时间 | 时间 | — |

### 4.2 题目（Question）

| 字段 | 类型 | 说明 |
|------|------|------|
| 题目ID | 主键 | 唯一标识 |
| 题目内容 | 富文本 | 题干，支持纯文本 |
| 题型 | 枚举 | 判断 / 单选 / 多选 |
| 选项 | JSON | [{label:"A", content:"选项内容"}, ...] |
| 正确答案 | 文本 | 判断："正确"/"错误"；单选："A"；多选："A,B" |
| 答案解析 | 富文本 | 解析说明 |
| 难度 | 枚举 | 简单 / 中等 / 困难 |
| 知识点标签 | 多选 | 如"地震灾害"/"洪涝灾害"等 |
| 权限状态 | 枚举 | public（公开）/ private（私有） |
| 来源 | 枚举 | platform（平台创建）/ province（省级创建） |
| 所属题库ID | 外键 | 关联题库 |
| 创建人 | 外键 | — |
| 创建时间 | 时间 | — |
| 更新时间 | 时间 | — |

**权限状态规则**：
- 仅 platform 来源的题目可设置权限状态
- 权限为 public 的题目，各省可在「从平台题库选用」页面中看到并选用
- 权限为 private 的题目，仅平台超管/运营可见，各省不可见
- province 来源的题目（省级自有题目）不设置权限状态，默认仅所属省可见

#### 题目创建/编辑页

**页面路径**：`/platform/question/form?id={questionId}`
**页面标题**：新建题目 / 编辑题目

**表单区块**：

| 字段 | 字段名 | 类型 | 必填 | 校验规则 | 说明 |
|------|--------|------|------|---------|------|
| 所属题库 | bankId | 下拉选择 | 是 | — | 平台共用题库列表 |
| 题型 | type | 单选 | 是 | — | 判断 / 单选 / 多选 |
| 题目内容 | content | 富文本 | 是 | 长度 10-500 | 题干 |
| 选项 | options | 动态表单 | 条件必填 | 单选/多选时必填，至少2个选项 | 选项标签（A/B/C/D）+ 选项内容 |
| 正确答案 | answer | 条件表单 | 是 | 判断：正确/错误；单选：单选选项；多选：多选选项 | 题型变化时动态变化 |
| 答案解析 | analysis | 富文本 | 否 | 长度 0-1000 | — |
| 难度 | difficulty | 单选 | 是 | — | 简单 / 中等 / 困难 |
| 知识点标签 | tags | 多选 | 否 | — | 从知识点字典中选择，支持搜索 |
| 权限状态 | permission | 单选 | 是 | 默认 private | 公开 / 私有 |

**选项字段交互规则**：
- 题型为"判断"时：不展示"选项"字段，正确答案字段变为两个单选按钮（正确 / 错误）
- 题型为"单选"时：选项字段展示 A/B/C/D 四个输入框，正确答案为单选按钮组
- 题型为"多选"时：选项字段展示 A/B/C/D/E（最少2个，最多6个，支持添加/删除），正确答案为多选框组

**页面操作按钮**：

| 按钮 | 触发条件 | 操作结果 | 成功提示 | 失败提示 |
|------|---------|---------|---------|---------|
| 保存 | 表单校验通过 | 保存题目，返回题目列表页 | 保存成功 | 显示具体错误 |
| 保存并继续添加 | 表单校验通过 | 保存题目，清空表单继续新建 | 保存成功 | 显示具体错误 |
| 取消 | 任意时刻 | 返回题目列表页，不保存 | — | — |

**校验规则**：
- 题目内容：必填，10-500字符
- 选项：单选/多选时必填，至少2个选项，选项内容不可为空
- 正确答案：必填，必须与题型匹配（单选只能选1个，多选至少选2个）
- 选项内容不可重复

### 5.3 题目批量导入页

**页面路径**：`/platform/question/import`
**页面标题**：批量导入题目

#### 导入流程

**步骤 1：下载模板**
- 提供 Excel 模板下载
- 模板字段：题型、题目内容、选项A、选项B、选项C、选项D、正确答案、答案解析、难度、知识点标签

**步骤 2：上传文件**
- 文件格式：.xlsx
- 大小限制：≤ 5MB
- 单次最多导入 500 条

**步骤 3：数据预览与校验**
- 表格展示解析后的数据
- 错误行高亮，显示错误原因（如题型不合法、选项不足、正确答案不匹配等）
- 仅全部正确时才可提交

**步骤 4：选择目标题库**
- 下拉选择：导入到哪个平台共用题库

**步骤 5：确认导入**
- 显示可导入条数
- 确认后批量写入

**导入规则**：
- 题型只允许：判断 / 单选 / 多选
- 判断题：选项A=正确，选项B=错误，正确答案填"正确"或"错误"
- 单选题：至少提供选项A、B，正确答案填选项标签（如"A"）
- 多选题：至少提供选项A、B，正确答案填多个选项标签（如"A,B"）
- 难度只允许：简单 / 中等 / 困难
- 知识点标签：如标签不存在，自动创建新标签

| 字段 | 字段名 | 类型 | 必填 | 校验规则 | 说明 |

| 关联题库 | bankId | 下拉选择 | 是 | — | 平台共用题库列表，确定后只能从该题库选题 |

**逐题选取模式**：
- 展示关联题库下的题目列表（仅 public 权限状态的题目）

**按条件批量选取模式**：
- 题型：多选（判断/单选/多选）
- 难度：多选（简单/中等/困难）
- 知识点：多选
- 抽取数量：数字输入，≤ 关联题库下符合条件的题目总数

| 按钮 | 触发条件 | 操作结果 |
|------|---------|---------|
| 保存 | 表单校验通过 | 保存题库包，返回列表页 |
| 取消 | 任意时刻 | 返回列表页，不保存 |

#### 题库包使用场景

1. **省级整包选用**：省级管理员在「从平台题库选用」页面，可切换"逐题选取"和"整包选用"两种模式。整包选用时，将题库包内所有题目一次性复制到省级自主题库。
2. **租户开通时装载**：运营在租户开通阶段，可直接为省级租户装载指定题库包，快速初始化省级题库。

- 复制后的题目可独立编辑，不影响平台原题

## 7. 题库与赛事的绑定关系

### 7.1 练习赛关联题库

- 练习赛统一关联平台共用题库，由平台运营在「练习赛规则配置」中指定
- 省级管理员不可修改练习赛的题库范围

### 7.2 正式赛事关联题库

- 省级管理员在创建/编辑竞赛时，从"本省题库列表"中选择关联题库
- 一个赛事只能关联一个题库
- 赛事发布后，系统从关联题库中生成题目快照（同卷机制）
- 赛事发布前，可更换关联题库；发布后不可更换

### 7.3 题库充足性检查

- 创建竞赛时，若关联题库的题目数量 < 赛事配置的题目数量，给出提示："题库题目数量不足，当前 X 题，需要 Y 题"
- 省级管理员需补充题目后方可发布
