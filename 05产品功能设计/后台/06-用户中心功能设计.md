## 2. 模块概述

用户中心是平台的基础底座，负责：
1. **身份映射**：建立本平台用户与国家级平台用户之间的可关联身份
2. **登录态管理**：支撑小程序端的登录流程和会话生命周期
3. **用户数据管理**：支撑管理后台的用户列表查询、详情查看和数据范围控制
4. **数据同步**：从国家级平台同步用户基础数据到本平台

**核心规则**：
- 用户基础数据以国家级平台为主数据源，本平台不面向外部自由注册
- 两个平台应尽量使用可关联的统一身份标识
- 数据范围权限下沉：省级查看本省，市/县查看本级及下级
- 练习赛对所有已接入的灾害信息员开放；正式赛事受本省开通状态和参赛范围限制

## 5. 登录与身份映射

### 5.1 登录流程

```
学员打开小程序
    │
    ▼
输入国家级平台账号密码
    │
    ▼
[协同网关] 向国家级平台发起身份校验请求
    │
    ├── 校验成功 ──► 国家级平台返回用户唯一标识 + 基础信息
    │                   │
    │                   ▼
    │               [用户中心] 查询是否存在该标识的映射记录
    │                   │
    │                   ├── 存在 ──► 更新同步时间，登录成功
    │                   │
    │                   └── 不存在 ──► 创建新用户 + 建立映射记录，登录成功
    │
    └── 校验失败 ──► 返回"账号或密码错误"
```

### 5.2 身份映射规则

| 场景 | 处理规则 |
|------|---------|
| 首次登录 | 使用国家级平台返回的用户标识创建本平台用户，建立映射记录 |
| 重复登录 | 根据用户标识查询已有映射，更新最后同步时间 |
| 国家级平台用户信息变更 | 下次登录时同步更新本平台用户数据（姓名、单位、在岗状态等） |
| 国家级平台不可用 | 返回"登录服务暂时不可用，请稍后重试"；不影响已登录用户的会话 |

### 5.3 同步数据范围

登录时从国家级平台同步的字段：
- 姓名
- 手机号
- 所属省份/市/县/乡镇（行政区划信息）
- 所属单位
- 在岗状态

**不同步的字段**：
- 国家级平台的学习记录、课程进度（由协同网关单独处理）
- 国家级平台的密码（本平台不存储用户密码）

## 9. 平台运营配置后台：学员数据同步

**页面路径**：`/platform/user/sync`
**页面标题**：学员数据同步

### 9.1 同步配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 自动同步开关 | 开关 | 开启 | 是否定时自动从国家级平台同步用户数据 |
| 同步频率 | 下拉选择 | 每日凌晨 2:00 | 每日一次 / 每 6 小时 / 每 12 小时 |
| 同步范围 | 下拉选择 | 全量 | 全量同步 / 增量同步（仅同步变更） |

### 9.2 手动同步

**操作按钮**：「立即同步」

**同步过程展示**：
- 同步开始时间
- 当前同步进度（如"已同步 1,234 / 5,000 人"）
- 预计剩余时间
- 同步状态：进行中 / 已完成 / 失败

### 9.3 同步日志

| 时间 | 同步类型 | 同步人数 | 新增人数 | 更新人数 | 失败人数 | 状态 |
|------|---------|---------|---------|---------|---------|------|
| yyyy-MM-dd HH:mm | 自动-全量 | 5,000 | 120 | 45 | 0 | 成功 |
| yyyy-MM-dd HH:mm | 手动-增量 | 1,200 | 0 | 80 | 2 | 部分失败 |

**失败处理**：
- 点击"失败人数"可查看失败明细（用户标识、失败原因）
- 支持对失败记录"重新同步"
