# 租户与数据配置中台 · 功能设计

## 1. 文档定位

本文档承接《00-产品功能清单》、《01-系统管理功能设计》和《06-用户中心功能设计》，将租户管理与数据配置中台模块的功能点展开为可直接指导原型设计和开发的页面级设计。

## 2. 模块概述

租户是平台中的独立客户空间，每个租户通常对应一个省级单位。租户管理覆盖租户从签约到上线的全生命周期。

**核心规则**：
- 每个租户对应一个省级行政区划
- 租户开通时自动初始化：组织架构（省-市-县-乡）、省级管理员角色、省级管理员账号
- 租户状态只有两种：已上线 / 已停用
  - 创建/保存租户后状态即为「已上线」
  - 服务到期或主动停用后状态变为「已停用」
  - 已停用的租户可恢复为「已上线」
- 已停用的租户不可创建新赛事，已有数据保留只读
- 平台运营/超管可代租户创建竞赛，标记为"平台代运营"

**开通初始化内容**：

| 初始化项 | 说明 | 来源 |
|---------|------|------|
| 租户空间 | 独立的租户标识和数据隔离边界 | 系统自动生成 |
| 组织架构 | 省-市-县-乡四级机构树 | 从行政区划数据导入或手动配置 |
| 省级管理员角色 | 预置角色，含赛事管理、题库管理、数据统计等权限 | 系统预置模板 |
| 省级管理员账号 | 初始管理员账号 | 运营手动创建或批量导入 |
| 平台共用题库 | 自动装载到租户空间 | 从平台题库复制 |

## 5. 租户状态

租户仅有两种状态，状态切换逻辑如下：

```
┌─────────┐   创建/保存    ┌─────────┐   到期/停用    ┌─────────┐
│   无   │ ────────────> │ 已上线  │ ───────────> │ 已停用  │
└─────────┘               └─────────┘              └─────────┘
                               ▲                        │
                               │                        │
                               └──────── 恢复 ──────────┘
```

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| 已上线 | 租户正常提供服务 | 编辑、停用 |
| 已停用 | 服务已暂停，数据保留只读 | 编辑、恢复 |

## 6. 租户列表页

**页面路径**：`/platform/tenant/list`
**页面标题**：租户管理
**数据对象**：租户（Tenant）

### 6.1 列表区块

**列定义**：

| 列名 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 序号 | — | 自增 | — |
| 租户名称 | name | 文本 | — |
| 对应省份 | provinceName | 文本 | — |
| 租户状态 | status | 标签 | 待开通（灰）/ 联调中（黄）/ 已上线（绿）/ 已停用（红） |
| 联系人 | contactName | 文本 | — |
| 联系人手机号 | contactPhone | 文本 | 脱敏展示 |
| 开通时间 | openTime | 时间 | yyyy-MM-dd |
| 有效期至 | expireTime | 时间 | yyyy-MM-dd |
| 操作 | — | 按钮组 | 根据状态动态展示 |

**筛选条件**：
- 租户名称：文本输入，模糊搜索
- 租户状态：下拉选择（已上线 / 已停用）
- 对应省份：下拉选择
- 开通时间：日期范围

**分页**：每页 10/20/50 条

**顶部操作按钮**：
- **开通租户**：跳转「租户开通页」

**行内操作按钮（按状态动态展示）**：

| 当前状态 | 可执行操作 |
|---------|-----------|
| 已上线 | 编辑 / 停用 |
| 已停用 | 编辑 / 恢复 |

### 6.2 租户状态批量操作

列表左侧支持 Checkbox 多选，勾选后顶部显示批量操作按钮：
- **批量停用**：仅对已上线租户生效
- **批量恢复**：仅对已停用租户生效

## 7. 租户开通页

**页面路径**：`/platform/tenant/create`
**页面标题**：开通租户

租户开通在一个页面内完成，运营人员填写基本信息、配置组织架构、创建初始管理员账号并设置赛事能力开关后提交，系统立即创建租户空间并初始化相关数据，租户状态直接变为「已上线」。

### 7.1 填写租户基本信息

| 字段 | 字段名 | 类型 | 必填 | 校验规则 | 说明 |
|------|--------|------|------|---------|------|
| 租户名称 | name | 文本 | 是 | 长度 2-100 | 如"浙江省应急管理厅" |
| 对应省份 | provinceCode | 下拉选择 | 是 | 不可重复选择已开通省份 | 从行政区划字典中选择 |
| 联系人姓名 | contactName | 文本 | 是 | — | — |
| 联系人手机号 | contactPhone | 文本 | 是 | 手机号格式 | — |
| 有效期至 | expireTime | 日期 | 否 | 默认 1 年后 | 租户服务到期时间 |

### 7.2 导入组织架构

**方式一：从行政区划导入**
- 系统自动根据对应省份的行政区划数据生成省-市-县-乡机构树
- 运营人员可预览并调整（删除不需要的节点、修改节点名称）

**方式二：手动配置**
- 逐条添加市/县/乡级节点
- 支持 Excel 批量导入（使用系统管理中的机构导入模板）

### 7.3 创建初始管理员账号

| 字段 | 字段名 | 类型 | 必填 | 说明 |
|------|--------|------|------|------|
| 账号 | username | 文本 | 是 | 登录账号，字母+数字，长度 4-20 |
| 姓名 | name | 文本 | 是 | — |
| 手机号 | phone | 文本 | 是 | — |
| 初始密码 | password | 密码 | 是 | 系统随机生成，开通后通过短信发送 |
| 角色 | roleId | 下拉选择 | 是 | 默认"省级管理员"，可选其他预置角色 |

**批量创建**：支持一次创建多个管理员账号（如省级管理员 + 若干市级管理员）。

### 7.4 配置赛事能力开关

**赛事能力开关**（练习赛、学习联动、数据统计、题库管理等通用功能默认全部开启，无需配置）：

| 赛事能力 | 开关 | 默认状态 |
|---------|------|---------|
| 初赛-个人赛 | 启用/停用 | 启用 |
| 初赛-团体赛 | 启用/停用 | 启用 |
| 复赛-个人赛 | 启用/停用 | 启用 |
| 复赛-团体赛 | 启用/停用 | 启用 |
| 决赛-个人赛 | 启用/停用 | 启用 |
| 决赛-团体赛 | 启用/停用 | 启用 |

**提交后自动执行**：
1. 创建租户空间
2. 生成机构树
3. 创建管理员账号
4. 初始化角色权限
5. 装载平台共用题库（复制平台题目到租户空间）
6. 租户状态变为「已上线」
