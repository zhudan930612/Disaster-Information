# 系统管理 · 功能设计

## 1. 文档定位

本文档承接《08-产品功能清单》，将系统管理模块的功能点展开为可直接指导原型设计和开发的页面级设计，包含字段定义、操作规则、状态流转和数据校验。

本文档面向下一阶段原型设计的 AI/设计师，需包含足够的字段、状态和规则信息，使其能直接输出可交互原型。

- 《08-产品功能清单》
- 《01应用架构与模块拆分》

## 2. 模块概述

系统管理是平台的权限与组织底座，支撑省-市-县-乡四级组织架构、后台角色权限、管理员账号、操作审计和基础配置。所有业务模块的数据权限范围（本级及下级）都依赖机构树和角色数据权限配置。

**包含子模块**：

| 子模块 | 优先级 | 说明 |
|--------|--------|------|
| 角色管理 | P0 | 配置后台角色及菜单/数据权限 |
| 账号管理 | P0 | 管理后台管理员账号 |
| 机构管理 | P0 | 管理省-市-县-乡组织架构树 |
| 日志管理 | P1 | 操作审计日志 + 协同同步日志 |
| 协议管理 | P2 | 用户协议与隐私政策配置 |
| 导航管理 | P2 | 后台菜单导航结构配置 |

## 4. 角色管理

### 4.1 角色列表页

**页面路径**：`/system/role/list`
**页面标题**：角色管理
**数据对象**：角色（Role）

#### 列表区块

**列定义**：

| 列名 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 序号 | — | 自增 | 列表序号 |
| 角色名称 | name | 文本 | 如"省级主管管理员" |
| 角色编码 | code | 文本 | 唯一标识，如"prov_admin" |
| 数据权限 | dataScope | 枚举 | 全部/本级及下级/仅本级/无 |
| 状态 | status | 枚举 | 启用/停用 |
| 创建时间 | createTime | 时间 | yyyy-MM-dd HH:mm |
| 操作 | — | 按钮组 | 编辑 / 停用 / 启用 / 删除 |

**筛选条件**：
- 角色名称：文本输入，模糊搜索
- 状态：下拉选择（全部/启用/停用）

**分页**：每页 10/20/50 条，底部翻页

**顶部操作按钮**：
- **新建角色**：跳转「角色创建/编辑页」

**行内操作按钮**：
- **编辑**：跳转「角色创建/编辑页」，携带 roleId
- **停用/启用**：切换角色状态，停用后该角色下账号无法登录
- **删除**：仅当该角色下无关联账号时可删除，二次确认弹窗

**空状态**：列表为空时显示"暂无角色数据" + "新建角色"引导按钮

#### 状态规则

- 角色状态为"停用"时，该角色下所有账号禁止登录
- 预置角色（系统初始自带）不可删除，可编辑权限范围
- 角色编码全局唯一，创建后不可修改

---

### 4.2 角色创建/编辑页

**页面路径**：`/system/role/form?id={roleId}`（编辑时携带id，新建时不携带）
**页面标题**：新建角色 / 编辑角色
**数据对象**：角色（Role）

#### 基础信息区块

| 字段 | 字段名 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|------|--------|------|------|--------|---------|------|
| 角色名称 | name | 文本 | 是 | — | 长度 2-20，不可重复 | 如"省级主管管理员" |
| 角色编码 | code | 文本 | 是 | — | 英文+数字+下划线，唯一，编辑时只读 | 如"prov_admin" |

#### 菜单权限配置区块

**功能描述**：以树形结构勾选该角色可访问的菜单页面

**展示形式**：树形选择器（CheckBox Tree）
- 一级节点：各功能模块（数据统计、题库、竞赛、用户管理等）
- 二级节点：具体页面（列表页、创建页、详情页等）
- 三级节点：页面内操作权限（查看/创建/编辑/删除/导出）
- 支持父节点全选、子节点独立勾选

**预置角色的默认菜单权限**：

| 角色 | 默认菜单权限 |
|------|-------------|
| 省级主管管理员 | 数据统计、题库、竞赛、用户管理（仅本省数据） |
| 地市县执行管理员 | 数据统计、用户管理（仅本级及下级数据） |
| 平台运营人员 | 租户、题库、练习赛规则、学习联动、用户管理 |
| 平台超级管理员 | 全部菜单 |
| 国家级管理员（只读） | 仅国家级数据看板（只读） |

#### 数据权限配置区块

| 字段 | 字段名 | 类型 | 必填 | 默认值 | 说明 |
|------|--------|------|------|--------|------|
| 数据权限范围 | dataScope | 单选 | 是 | 仅本级 | 决定该角色能查看的数据范围 |

**数据权限范围选项**：

| 选项 | 说明 | 适用角色 |
|------|------|---------|
| 全部 | 可查看全平台所有数据 | 平台超管、平台运营 |
| 本级及下级 | 可查看本级及所有下级机构数据 | 省级主管管理员、地市县执行管理员 |
| 仅本级 | 仅可查看本级机构数据 | — |
| 无 | 不可查看任何业务数据 | — |

**数据权限与机构树的关联规则**：
- "本级及下级"的判定依赖账号绑定的机构节点
- 例如：账号绑定到"杭州市"，则"本级及下级" = 杭州市 + 其下所有区县/乡镇
- 账号绑定到"浙江省"，则"本级及下级" = 浙江省 + 其下所有地市/区县/乡镇

#### 页面操作按钮

| 按钮 | 触发条件 | 操作结果 | 成功提示 | 失败提示 |
|------|---------|---------|---------|---------|
| 保存 | 表单校验通过 | 保存角色，返回列表页 | 保存成功 | 显示具体错误 |
| 取消 | 任意时刻 | 返回列表页，不保存 | — | — |

**校验规则**：
- 角色名称：必填，2-20字符，不可与现有角色重复
- 角色编码：必填，英文/数字/下划线，不可与现有编码重复
- 至少勾选 1 个菜单权限

## 5. 账号管理

### 5.1 账号列表页

**页面路径**：`/system/account/list`
**页面标题**：账号管理
**数据对象**：后台管理员账号（AdminAccount）

#### 列表区块

**列定义**：

| 列名 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 序号 | — | 自增 | 列表序号 |
| 账号名 | username | 文本 | 登录账号 |
| 姓名 | realName | 文本 | 真实姓名 |
| 手机号 | phone | 文本 | 脱敏展示，如 138****1234 |
| 所属角色 | roleName | 文本 | 关联角色的名称 |
| 所属机构 | orgName | 文本 | 绑定的机构节点名称 |
| 状态 | status | 枚举 | 启用/停用 |
| 创建时间 | createTime | 时间 | yyyy-MM-dd HH:mm |
| 操作 | — | 按钮组 | 编辑 / 重置密码 / 停用 / 启用 |

**筛选条件**：
- 账号名/姓名：文本输入，模糊搜索
- 所属角色：下拉选择（全部角色列表）
- 所属机构：树形选择器（机构树）
- 状态：下拉选择（全部/启用/停用）

**分页**：每页 10/20/50 条

**顶部操作按钮**：
- **新建账号**：跳转「账号创建/编辑页」

**行内操作按钮**：
- **编辑**：跳转「账号创建/编辑页」，携带 accountId
- **重置密码**：点击后弹出确认弹窗，确认后生成随机密码并展示（仅展示一次），同时发送短信通知
- **停用/启用**：切换账号状态

**空状态**："暂无账号数据" + "新建账号"引导按钮

#### 状态规则

- 账号状态为"停用"时，该账号无法登录后台
- 每个账号必须绑定一个角色和一个机构节点
- 账号的"数据权限范围"继承自所属角色的 dataScope 配置

---

### 5.2 账号创建/编辑页

**页面路径**：`/system/account/form?id={accountId}`
**页面标题**：新建账号 / 编辑账号
**数据对象**：后台管理员账号（AdminAccount）

#### 基础信息区块

| 字段 | 字段名 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|------|--------|------|------|--------|---------|------|
| 账号名 | username | 文本 | 是 | — | 长度 4-20，英文/数字，全局唯一 | 登录账号 |
| 姓名 | realName | 文本 | 是 | — | 长度 2-20 | 真实姓名 |
| 手机号 | phone | 手机号 | 是 | — | 11位手机号，全局唯一 | 用于接收通知 |
| 所属角色 | roleId | 下拉选择 | 是 | — | — | 角色列表，仅展示"启用"状态的角色 |
| 所属机构 | orgId | 树形选择 | 是 | — | — | 机构树，单选 |
| 状态 | status | 单选 | 是 | 启用 | — | 启用/停用 |

**新建时额外字段**：

| 字段 | 字段名 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|------|--------|------|------|--------|---------|------|
| 初始密码 | password | 密码 | 是 | 随机生成 | 长度 8-20，含字母+数字 | 支持自动生成或手动输入 |
| 确认密码 | confirmPassword | 密码 | 是 | — | 与密码一致 | — |

**编辑时规则**：
- 账号名不可修改（只读展示）
- 密码不展示，提供"重置密码"按钮（同列表页逻辑）

**机构树选择器交互**：
- 点击字段弹出树形浮层
- 树形展示省-市-县-乡四级机构
- 支持搜索机构名称快速定位
- 单选，选中后回显机构完整路径（如：浙江省 > 杭州市 > 西湖区）

#### 页面操作按钮

| 按钮 | 触发条件 | 操作结果 | 成功提示 | 失败提示 |
|------|---------|---------|---------|---------|
| 保存 | 表单校验通过 | 保存账号，返回列表页 | 保存成功，初始密码为：xxxx（新建时） | 显示具体错误 |
| 取消 | 任意时刻 | 返回列表页，不保存 | — | — |

**校验规则**：
- 账号名：必填，4-20字符，英文/数字，不可重复
- 姓名：必填，2-20字符
- 手机号：必填，11位合法手机号，不可重复
- 所属角色：必填
- 所属机构：必填
- 初始密码（新建时）：必填，8-20字符，至少包含字母和数字
- 确认密码：必填，与密码一致

## 6. 机构管理

### 6.1 机构树列表页

**页面路径**：`/system/org/tree`
**页面标题**：机构管理
**数据对象**：机构（Organization）

#### 树形列表区块

**展示形式**：左侧树形面板 + 右侧详情面板（或纯树形表格）

**树形节点展示字段**：

| 字段 | 说明 |
|------|------|
| 机构名称 | 如"浙江省"、"杭州市" |
| 机构编码 | 如"330000"、"330100" |
| 机构类型 | 省/市/县/乡 |

**树形操作按钮（每节点行内）**：
- **添加下级**：在该节点下新增子机构，跳转「机构创建/编辑页」，上级机构自动填充
- **编辑**：跳转「机构创建/编辑页」
- **停用/启用**：切换机构状态
- **删除**：仅当该机构下无子机构且无关联账号时可删除，二次确认

**顶部操作按钮**：
- **新增根节点**：新增省级机构（通常平台初始化时已有根节点，此按钮对超管显示）
- **批量导入**：跳转「机构导入页」
- **展开全部 / 折叠全部**：树形控制

**搜索功能**：
- 顶部搜索框：输入机构名称，树形自动定位并高亮匹配节点

#### 状态规则

- 机构状态为"停用"时，该机构下所有账号无法登录
- 删除机构时，需校验：无子机构、无关联账号、无关联用户（灾害信息员学员）
- 机构编码在同级下唯一，建议采用行政区划编码规则

### 6.2 机构创建/编辑页

**页面路径**：`/system/org/form?id={orgId}&parentId={parentId}`
**页面标题**：新建机构 / 编辑机构
**数据对象**：机构（Organization）

#### 基础信息区块

| 字段 | 字段名 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|------|--------|------|------|--------|---------|------|
| 上级机构 | parentId | 树形选择 | 是 | 传入的 parentId | — | 编辑时不可修改上级 |
| 机构名称 | name | 文本 | 是 | — | 长度 2-50 | 如"杭州市" |
| 机构编码 | code | 文本 | 是 | — | 英文/数字，同级下唯一 | 如"330100" |
| 机构类型 | type | 单选 | 是 | 自动推断 | 不可修改 | 省/市/县/乡，根据上级机构层级自动推断 |
| 排序号 | sort | 数字 | 否 | 0 | 整数，升序排列 | 控制同级机构展示顺序 |
| 状态 | status | 单选 | 是 | 启用 | — | 启用/停用 |

**机构类型推断规则**：
- 上级为空 → 类型 = 省
- 上级为省 → 类型 = 市
- 上级为市 → 类型 = 县
- 上级为县 → 类型 = 乡
- 上级为乡 → 不允许添加下级

#### 页面操作按钮

| 按钮 | 触发条件 | 操作结果 | 成功提示 | 失败提示 |
|------|---------|---------|---------|---------|
| 保存 | 表单校验通过 | 保存机构，返回树列表页 | 保存成功 | 显示具体错误 |
| 取消 | 任意时刻 | 返回树列表页，不保存 | — | — |

**校验规则**：
- 机构名称：必填，2-50字符
- 机构编码：必填，英文/数字，同级下不可重复
- 同一父节点下，机构名称不可重复
- 最多支持 4 级（省-市-县-乡），乡级节点不允许再有子节点

### 6.3 机构导入页

**页面路径**：`/system/org/import`
**页面标题**：批量导入机构
**数据对象**：机构（Organization）

#### 导入流程

**步骤 1：下载模板**
- 提供 Excel 模板下载按钮
- 模板字段：上级机构编码、机构编码、机构名称、机构类型、排序号

**步骤 2：上传文件**
- 文件格式：.xlsx
- 文件大小限制：≤ 2MB
- 支持拖拽上传

**步骤 3：数据预览与校验**
- 展示解析后的数据列表
- 错误行高亮标注，显示错误原因（如编码重复、上级不存在、层级超限等）
- 仅允许导入"全部正确"的数据，有错则无法提交

**步骤 4：确认导入**
- 显示可导入条数
- 点击"确认导入"后批量写入

**导入规则**：
- 上级机构编码必须在系统中已存在（根节点除外）
- 机构编码全局唯一
- 导入的机构类型必须与层级关系一致（如上级是市，则下级必须是县）
- 不覆盖已有数据，编码重复时报错

## 7. 日志管理

### 7.1 操作日志列表页

**页面路径**：`/system/log/operation`
**页面标题**：操作日志
**数据对象**：操作日志（OperationLog）

#### 列表区块

**列定义**：

| 列名 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 序号 | — | 自增 | 列表序号 |
| 操作人 | operatorName | 文本 | 执行操作的管理员姓名 |
| 所属租户 | tenantName | 文本 | 操作人所属的租户（如适用） |
| 操作模块 | module | 文本 | 如"竞赛管理"、"题库管理" |
| 操作类型 | actionType | 枚举 | 创建/编辑/删除/导出/登录/登出 |
| 操作内容 | content | 文本 | 简要描述，如"创建竞赛：西湖区灾害信息员大赛" |
| 操作时间 | operationTime | 时间 | yyyy-MM-dd HH:mm:ss |
| IP 地址 | ipAddress | 文本 | 操作人登录IP |

**筛选条件**：
- 操作人：文本输入，模糊搜索
- 操作模块：下拉选择（全部模块列表）
- 操作类型：下拉多选（创建/编辑/删除/导出/登录/登出）
- 操作时间：日期范围选择器

**分页**：每页 20/50/100 条

**顶部操作按钮**：
- **导出日志**：导出当前筛选条件下的日志为 Excel

**状态规则**：
- 操作日志只读，不可编辑、不可删除
- 日志保留期限：默认 180 天，超期自动清理（可配置）

### 7.2 协同日志列表页

**页面路径**：`/system/log/sync`
**页面标题**：协同日志
**数据对象**：协同同步日志（SyncLog）

#### 列表区块

**列定义**：

| 列名 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 序号 | — | 自增 | 列表序号 |
| 同步类型 | syncType | 枚举 | 用户数据同步/学习状态同步/次数发放/结果回流 |
| 同步方向 | direction | 枚举 | 国家级平台→本平台 / 本平台→国家级平台 |
| 状态 | status | 枚举 | 成功/失败/重试中 |
| 关联用户 | userName | 文本 | 涉及的用户姓名（如适用） |
| 请求内容 | requestContent | 文本 | 请求摘要，可点击展开查看完整 JSON |
| 响应内容 | responseContent | 文本 | 响应摘要，可点击展开查看完整 JSON |
| 失败原因 | errorMsg | 文本 | 失败时展示错误信息 |
| 同步时间 | syncTime | 时间 | yyyy-MM-dd HH:mm:ss |

**筛选条件**：
- 同步类型：下拉选择
- 状态：下拉选择（全部/成功/失败/重试中）
- 同步时间：日期范围选择器

**行内操作按钮**：
- **重试**（仅失败状态显示）：手动触发该条同步任务重试
- **查看详情**：弹窗展示完整的请求/响应 JSON
