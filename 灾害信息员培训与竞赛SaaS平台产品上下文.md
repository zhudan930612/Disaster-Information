## 产品定位

**系统名称**：灾害信息员大赛平台  
**产品形态**：独立部署的微信小程序 + 管理后台 + 平台运营配置后台

**核心定位**：本系统不是一个完整重做的“培训与竞赛一体平台”，而是一个面向省级客户的独立竞赛平台。平台重点承接练习赛、正式赛事、赛事组织、排名晋级、竞赛统计等能力；课程学习、视频学习、学习记录等培训能力继续由现有国家级培训竞赛平台承接，两套系统通过数据互通和业务协同形成“以赛促学”的联动闭环。

**国家级平台与本平台的边界**：

- **国家级培训竞赛平台（外部协同平台）**
  - 承接课程学习、视频播放、学习记录、学习任务完成状态、培训成长能力
  - 作为用户基础数据主数据源，向本平台同步姓名、省份、所属单位、在岗状态等基础信息，并提供学习激励联动所需的数据基础
- **本平台（当前建设对象）**
  - 承接练习赛、正式比赛、个人赛、团体赛、赛事运营、比赛统计、赛事结果沉淀
  - 独立部署、独立运行，但与国家级平台进行数据互通和业务协同

**系统关系边界**：

- 本平台是独立微信小程序，不是国家级平台的一个页面或子页面。
- 本平台与国家级平台代码、部署、运行环境独立。
- 本平台不承担课程学习主能力，学习能力由国家级平台提供。
- 本平台基于国家级平台回传的学习任务完成状态，直接发放额外练习赛答题次数等竞赛侧激励。
- 本平台中的正式赛事、练习赛、排名、晋级、赛事统计由本平台独立负责。
- 国家级管理员在本系统中仅承担全国/省级汇总数据以及授权范围内部分租户级明细的查看职责，不参与赛事开通、配置和日常管理。

**学习激励联动边界**：

- 本平台不直接提供视频课程播放。
- 当学员在本平台练习赛中答题次数不足时，可引导其前往国家级平台完成指定学习任务。
- 额外答题次数应基于“完成指定学习任务”发放，而不是简单“打开视频即发放”。
- 指定学习任务可包括：完成指定课程、达到最低学习时长、完成课后小测中的一种或组合。
- 学习任务完成后，国家级平台向本平台同步完成状态和可发放次数，本平台据此直接增加用户可用答题次数。

**账号与访问边界**：

- 原则上不面向外部自由注册，面向所有已接入的灾害信息员用户开放，用户基础数据以国家级平台为主数据源同步到本平台。
- 两个平台应尽量使用可关联的统一身份标识，确保同一用户在学习侧和竞赛侧可被准确识别。
- 练习赛对所有已接入的灾害信息员用户可用；正式赛事仍受本省开通状态和本省权限限制。
- 用户默认只能参加本省已开通的正式赛事。

**目标用户**：

- **省级主管管理员**：组织本省初赛、复赛、决赛，管理赛事配置，查看赛事统计
- **地市县执行管理员**：落实参赛组织、名单管理、过程督导、结果查看
- **灾害信息员学员**：参加练习赛、正式比赛，并通过外部学习任务获取更多练习机会
- **国家级管理员（只读）**：查看全国/省级汇总数据和授权范围内部分租户级明细，不参与管理操作
- **平台运营与实施团队（我方）**：完成租户开通、数据配置、赛事配置、题库装载和运营支持
- **平台超级管理员**：负责跨租户配置、安全、审计和系统级能力治理

**核心场景**：

- **练习赛场景**：免费开放给所有已接入的灾害信息员用户进行日常答题训练，答题次数可与国家级平台学习任务联动
- **正式赛事场景**：组织个人赛、团体赛和省级初赛、复赛、决赛
- **赛事监管场景**：查看报名、参赛、成绩、排名、晋级等赛事统计结果
- **学习激励场景**：练习次数耗尽后，引导到国家级平台完成学习任务并回流次数
- **SaaS 交付场景**：省级客户签约后，快速完成租户开通、数据配置和赛事上线

## 商业模式

**商业模式定位**：

- **公司定位**：应急管理数字化场景的解决方案与平台服务商
- **核心产品**：省级灾害信息员大赛平台能力输出
- **核心客户**：省级应急管理相关单位及其下属管理体系
- **业务聚焦**：标准化赛事能力、独立微信小程序、低成本配置化交付、与国家级平台协同联动

**收费模式**：

- **省级客户**：以竞赛平台服务采购为主，采购后开通本省正式赛事和统计能力
- **竞赛服务能力**：随租户开通生效，包含省级初赛、复赛、决赛组织、团体赛组织与统计能力
- **增值服务**：题库共建、赛事保障、专项运营活动、数据分析支持

**核心价值主张**：

- **对省级主管部门**：快速具备可组织、可监管、可量化的线上赛事抓手
- **对地市县执行单位**：减少线下组织压力，提升名单管理、赛事执行和结果统计效率
- **对灾害信息员学员**：有独立竞赛入口，可练可赛，并能通过国家级平台学习获得额外练习机会
- **对我方运营**：通过标准化竞赛平台和配置化交付实现跨省复制

**战略原则**：

- 竞赛能力由本平台统一建设
- 学习能力优先复用国家级平台，不在本平台重复建设
- 省级差异通过“数据与规则配置”吸收，而不是重定制开发

**核心竞争力与商业壁垒**：

- 国家级平台建设经验带来的业务理解和内容经验
- 省级竞赛场景的模板化产品能力
- 学习与竞赛联动机制带来的更强活跃和参赛转化

**风险规避策略**：

- 避免在本平台内重复建设成熟学习能力
- 保持赛事规则统一，避免租户规则分裂
- 对跨系统同步设置兜底机制，避免国家级平台接口异常导致本平台主链路失效

**对功能设计的影响**：

- 所有功能优先围绕“竞赛主链路”展开，而不是扩展为完整学习平台
- 与国家级平台的协同设计必须视为核心能力，不是边缘接口

## 术语表

| 术语 | 定义 | 易混淆概念 |
|------|------|------------|
| 国家级平台 | 已上线的国家级培训竞赛平台，承担学习侧能力 | 不是本系统内部模块，而是外部协同平台 |
| 大赛平台 | 本次建设的独立微信小程序及后台系统 | 不是“培训一体平台” |
| 练习赛 | 面向学员的常态化答题训练能力 | 与正式比赛不同，不直接决定晋级 |
| 正式比赛 | 按赛制规则开展的正式赛事过程 | 与日常练习赛不同 |
| 团体赛 | 按县、市、省等层级组队进行的赛事 | 与个人赛并行，不可替代 |
| 学习激励任务 | 在国家级平台中定义的学习任务，完成后可直接触发本平台额外练习次数发放 | 不是本平台自建课程任务 |
| 奖励答题次数 | 学员完成指定学习任务后获得的额外练习赛次数 | 不建议定义为“单纯看视频即发放” |
| 赛制模板 | 平台预置的标准赛事机制模板，省级按开关和参数配置启用 | 不等同于自定义规则引擎 |
| 租户 | 平台中的独立客户空间，通常对应省级单位 | 与行政层级不同 |
| 数据配置 | 通过配置完成组织、人员、题库、赛事和协同策略上线准备 | 与定制开发不同 |

## 模块清单

### 竞赛小程序端（首页/练习/赛事/我的）

**定位**：面向灾害信息员学员的独立参赛与练习入口。  
**核心场景**：练习赛、正式比赛、排行展示、学习任务联动提示、个人记录查看。  
**使用角色**：灾害信息员学员。  
**主要流程**：

```text
登录小程序 → 进入练习赛/正式比赛 → 消耗答题次数 → 查看成绩/排行/晋级状态 → 次数不足时跳转国家级平台完成学习任务 → 获取额外次数后返回继续使用
```

**与其他模块的关系**：

- ← 竞赛题库与模板中台：提供试题与练习赛模板
- ← 数据配置中台：提供可参与赛事、次数规则、门槛配置
- ← 国家级平台协同模块：提供学习任务完成状态和额外次数回流
- → 赛事统计模块：沉淀练习与比赛结果

### 竞赛管理后台

**定位**：面向省级与地市县管理员的赛事组织与赛事管理后台。  
**核心场景**：赛事发布、赛程配置、报名与资格管理、结果查看、排名查看。  
**使用角色**：省级主管管理员、地市县执行管理员。  
**主要流程**：

```text
创建赛事 → 选择模板 → 配置时间、题量、门槛、范围 → 发布赛事 → 查看参赛与结果 → 导出统计
```

**与其他模块的关系**：

- ← 数据配置中台：提供组织、人员、租户范围
- ← 竞赛题库与模板中台：提供比赛供题能力
- ← 国家级平台协同模块：获取学习任务完成状态用于资格或次数奖励
- → 赛事统计模块：输出比赛结果

### 赛事统计与结果模块

**定位**：沉淀赛事过程数据并支撑监管、复盘和通报。  
**核心场景**：参赛率、成绩分布、排行结果、晋级结果、练习数据统计。  
**使用角色**：省级主管管理员、地市县执行管理员、平台运营人员。  
**主要流程**：

```text
接收练习与比赛数据 → 形成统计看板与明细 → 输出结果与复盘信息
```

### 国家级平台协同模块

**定位**：承接本平台与国家级平台之间的数据互通与业务联动。  
**核心场景**：身份关联、学习任务跳转、学习完成状态回流、额外答题次数同步、比赛结果回流。  
**使用角色**：系统、平台运营人员、平台超级管理员。  
**主要流程**：

```text
读取用户身份映射 → 生成学习任务引导 → 接收国家级平台完成状态 → 换算并发放额外答题次数 → 回写用户可用次数和联动记录
```

**稳定规则**：

- 额外答题次数应基于“指定学习任务完成”发放
- 应支持每日上限、任务上限、次数有效期等控制
- 同步失败时应有兜底提示和补偿机制

### 多租户与租户运营模块

**定位**：支撑平台面向多个省级客户开通、配置和运行。  
**核心场景**：租户开通、生命周期管理、开通状态、协同策略配置。  
**使用角色**：我方运营人员、平台超级管理员。

### 数据配置中台

**定位**：吸收各省差异并降低交付成本。  
**核心场景**：行政区划、组织层级、人员导入、题库包、练习赛模板、正式赛事规则、答题次数规则、学习联动规则配置。  
**使用角色**：我方运营人员、省级管理员（部分配置权限）。

### 竞赛题库与模板中台

**定位**：统一管理竞赛题库、练习题库和练习赛模板。  
**核心场景**：题目入库、标签分类、组题策略、模板配置、题库装载。  
**使用角色**：内容运营、我方运营人员、省级管理员（部分权限）。

## 数据底座

> 平台数据底座以“人员组织、竞赛题库、练习与比赛过程、协同联动、租户配置”五类数据为核心。

### 人员与组织信息库

- 租户信息：租户 ID、状态、开通时间、开通能力
- 组织信息：省、市、县、乡镇等层级结构
- 人员信息：用户身份、所属地区、角色、在岗状态、来源系统标识，主数据来源为国家级平台

### 竞赛题库与模板资产库

- 题库资源：题型、难度、知识点、版本、归属
- 模板资源：练习赛模板
- 组题规则：练习赛组题、正式比赛组题

### 练习与比赛过程库

- 练习记录：答题次数、成绩、正确率、错题记录、排行快照
- 比赛记录：场次、轮次、成绩、晋级状态、排名结果
- 用户权益：剩余答题次数、奖励次数、次数有效期

### 外部学习协同数据域

- 身份映射：本平台用户 ID 与国家级平台用户标识映射
- 学习任务配置：任务 ID、任务类型、可奖励次数、限制条件
- 学习完成记录：完成状态、完成时间、可发放次数
- 协同日志：跳转记录、同步记录、失败原因、补偿状态

### 租户配置数据域

- 赛事开通：开通能力、有效期、正式赛事启用状态
- 配置快照：组织配置、题库配置、答题次数规则、协同规则

### 数据互通规则

- 国家级平台向本平台同步用户基础数据、学习任务完成状态与可发放次数
- 本平台向国家级平台回流完整比赛结果及相关成长数据
- 两平台对用户身份采用可关联的统一标识

### 数据协同价值

- 形成“学习在国家级平台，练习与比赛在本平台”的联动闭环
- 形成“配置 - 执行 - 统计 - 复盘”的赛事运营闭环
- 为省级客户提供可量化、可复盘的赛事管理依据

## 设计原则

- **竞赛主链路优先**：本平台首先服务练习赛和正式赛事，不扩展为完整学习平台
- **学习能力外部协同**：课程和视频学习优先复用国家级平台，不在本平台重复建设
- **任务完成优先于纯观看**：奖励答题次数应基于完成指定学习任务，不应简单按打开视频发放
- **配置优先于定制**：优先通过配置吸收省级差异，避免规则分裂
- **练习赛模板化、正式赛事规则标准化优先**：省级只能在固定边界内配置，不开放自由规则编排
- **租户隔离刚性**：跨租户数据、权限、日志严格隔离
- **跨系统协同可追溯**：所有跳转、回流、发放、失败补偿都必须有记录
- **结果可量化**：练习、比赛、排行、晋级、组织执行均需可统计、可复盘
- **移动端轻量优先**：面向基层学员的操作尽量短路径、即时反馈

## 功能规划原则

### 优先级判断标准

1. **赛事刚性要求** > **跨系统协同稳定性** > **租户隔离与配置化** > **体验优化**
2. 先打通竞赛主链路，再补强增值玩法
3. 先保证“能赛、能统、能联动”，再考虑“更好玩、更丰富”

### 必须优先具备的能力

- 独立竞赛小程序端
- 练习赛与正式比赛主链路
- 练习赛模板与正式赛事规则配置
- 答题次数与学习联动机制
- 用户身份映射与同步
- 赛事统计与结果沉淀
- 多租户与数据配置中台

### 后续按价值评估的能力

- 更丰富的激励玩法
- 更细的成长展示
- 智能推荐和智能运营辅助
- 与更多外部系统的深度协同

## 对接系统

| 系统名称 | 数据方向 | 交换内容 | 影响说明 |
|----------|----------|----------|----------|
| 国家级培训竞赛平台 | 双向 | 用户基础数据、学习任务配置、学习完成状态、可发放答题次数、完整比赛结果回流 | 影响“以赛促学”闭环是否成立，也影响账号和权限口径是否统一 |
| 全国灾害信息员数据库 / 报灾系统人员库（可选校验源） | 可选接收/校验 | 人员身份、在岗信息、组织归属 | 作为补充校验源，影响账号开通和权限准确性 |
| 短信/消息服务 | 下发 | 赛事通知、晋级通知、同步结果通知 | 影响触达效率 |
| 统计报表导出与归档系统（待确认） | 输出 | 比赛统计、排名结果、组织通报材料 | 影响汇报和归档规范 |

## 待确认事项（供后续产品规划同步）

- 国家级平台与本平台之间的账号打通方式：单点登录、绑定关系还是静默映射
- 学习任务完成的认定口径：课程完成、时长达标、小测通过中的哪种或哪几种
- 奖励答题次数的发放规则：单次奖励、每日上限、总上限、有效期
- 跨系统同步频率：实时、准实时还是定时同步
