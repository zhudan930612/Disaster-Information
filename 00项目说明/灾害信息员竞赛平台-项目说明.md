# 灾害信息员大赛平台 · 项目说明

> 本文档是 AI 理解本项目的入口文档。阅读本文档后，AI 应能掌握项目的全貌、核心边界、关键规则和各模块职责，从而具备开展后续设计、分析、问答等工作的基础。

---

## 1. 项目一句话概述

**灾害信息员大赛平台**是一个面向省级客户独立部署的竞赛 SaaS 平台（独立微信小程序 + 管理后台 + 运营配置后台），重点承接灾害信息员的练习赛、正式赛事（初赛/复赛/决赛）、赛事统计与排名晋级；课程学习、视频学习等培训能力由现有国家级培训竞赛平台承接，两套系统通过数据互通形成"以赛促学"的联动闭环。

---

## 2. 产品定位与边界

### 2.1 核心定位

- **不是** "培训与竞赛一体平台"
- **是** "独立竞赛平台" + "与国家级平台协同联动"

### 2.2 系统边界

| 能力 | 本平台（当前建设对象） | 国家级平台（外部协同） |
|------|------------------------|------------------------|
| 练习赛 | 承担 | — |
| 正式比赛（初赛/复赛/决赛） | 承担 | — |
| 团体赛 | 承担 | — |
| 赛事运营、排名、晋级、统计 | 承担 | — |
| 课程学习、视频播放 | — | 承担 |
| 学习任务定义与完成判定 | — | 承担 |
| 用户基础数据主数据 | — | 承担（同步到本平台） |

### 2.3 关键边界规则

- 本平台是**独立微信小程序**，不是国家级平台的子页面
- 本平台**不直接提供视频课程播放**
- 练习赛对所有已接入的灾害信息员用户**免费开放**
- 正式赛事仅面向**本省已开通**的用户开放
- 国家级管理员在本系统中**只读查看**，不参与赛事管理
- 学习激励联动：学员完成国家级平台指定学习任务 → 国家级平台同步完成状态 → 本平台直接发放额外练习次数

## 3. 用户角色

| 角色 | 核心职责 | 权限范围 |
|------|----------|----------|
| **灾害信息员学员** | 参加练习赛、正式比赛，完成学习联动任务 | 练习、参赛、查看排行和个人记录 |
| **省级主管管理员** | 组织本省赛事，配置规则，查看统计 | 本省赛事配置、发布、统计查看 |

## 4. 产品形态与终端

平台由三个前端终端组成：

1. **学员竞赛小程序（微信小程序）**：面向灾害信息员学员，练习/参赛/排行/次数/学习联动跳转
2. **竞赛管理后台（Web）**：面向省级/地市县管理员和国家级管理员，赛事管理/名单/统计/结果
3. **平台运营配置后台（Web）**：面向我方运营和平台超管，租户开通/题库/模板/规则/运维保障

## 5. 核心业务流程

### 流程一：练习赛与学习激励联动闭环

```
学员进入练习赛 → 消耗答题次数 → 次数充足则继续练习 → 次数不足则提示前往国家级平台
→ 跳转国家级平台完成指定学习任务 → 国家级平台同步完成状态
→ 本平台发放额外答题次数 → 学员返回继续练习
```

### 流程二：正式赛事闭环

```
开通省级赛事服务 → 配置正式赛事规则（时间/题量/时长/门槛/范围）→ 发布赛事
→ 用户报名/获得资格 → 进入初赛/复赛/决赛（个人赛或团体赛）
→ 记录成绩 → 计算排名与晋级 → 发布结果 → 统计与复盘
```

### 流程三：SaaS 交付闭环

```
客户签约 → 创建租户空间 → 初始化组织架构与角色
→ 导入组织/人员/地区数据 → 装载题库与配置赛事规则
→ 配置答题次数与学习联动规则 → 联调验证 → 租户上线
```

## 8. 关键产品规则

### 8.1 赛制规则

- 正式赛事固定按 **初赛 → 复赛 → 决赛** 三阶段流转
- 赛事机制采用**规则配置 + 参数调整**，不开放自由赛制编排
- 省级可配置：时间、题量、时长、晋级比例、题库范围、参赛门槛
- 省级不可配置：底层计分算法、排名算法、晋级链路逻辑、防作弊规则

### 8.2 题库规则

- 仅支持 **判断、单选、多选** 三种题型
- 题库分层：**平台共用题库** + **省级私有题库**
- 组题策略化：省级**不能逐题手工指定**，只能在策略参数边界内配置
- 正式赛事采用**同卷机制**：发布后、开赛前锁定题目集合，所有参赛者答同一套题
- 练习赛每次独立随机抽题，支持**去重机制**（近期已答题目回避）
- 题目变更通过**版本管理**隔离，不影响历史比赛复盘

### 8.3 答题次数规则

- 次数分两类：**基础次数**（每日发放，默认 3 次）+ **奖励次数**（完成学习任务发放）
- 扣减顺序：先基础次数，后奖励次数
- 次数默认**当日有效，次日清零**
- 奖励次数需支持：单次奖励量、每日上限、累计上限、有效期配置

### 8.4 学习联动规则

- 采用"完成指定学习任务后**直接发放**额外练习次数"机制
- 学习任务完成口径：**完成指定课程**（由国家级平台判定）
- 不推荐"打开视频即发放"的弱规则
- 同步失败需有兜底提示（"待到账/稍后重试"）和后台补偿机制
- 国家级平台不可用时，不影响本平台主链路运行

### 8.5 多租户与权限规则

- 一个省级客户 = 一个独立租户
- 租户数据（组织、人员、赛事、结果、日志）**严格隔离**
- 地市县管理员只能查看**本级及下级**范围
- 国家级管理员**只读**，不参与赛事配置和管理
- 所有关键操作（创建/编辑/发布赛事、发布结果、导出、人工审核）必须**留痕**

## 10. 设计原则

1. **竞赛主链路优先**：本平台首先服务练习赛和正式赛事，不扩展为完整学习平台
2. **学习能力外部协同**：课程和视频学习复用国家级平台，不在本平台重复建设
3. **任务完成优先于纯观看**：奖励次数基于完成指定学习任务，不简单按打开视频发放
4. **配置优先于定制**：通过配置吸收省级差异，避免规则分裂
5. **租户隔离刚性**：跨租户数据、权限、日志严格隔离
6. **跨系统协同可追溯**：所有跳转、回流、发放、失败补偿都必须有记录
7. **结果可量化**：练习、比赛、排行、晋级、组织执行均需可统计、可复盘
8. **移动端轻量优先**：面向基层学员的操作短路径、即时反馈

## 12. 术语表

| 术语 | 定义 |
|------|------|
| 国家级平台 | 已上线的国家级培训竞赛平台，承担学习侧能力（外部协同平台） |

| 租户 | 平台中的独立客户空间，通常对应一个省级单位 |
| 同卷机制 | 正式赛事中，所有参赛者使用相同的题目集合 |
| 数据配置 | 通过配置完成组织、人员、题库、赛事和协同策略上线准备 |
| 基础次数 | 平台每日提供的默认练习次数 |
| 奖励次数 | 完成学习任务后发放的额外练习次数 |
